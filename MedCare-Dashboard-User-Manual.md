# คู่มือการใช้งาน MedCare Dashboard

เอกสารฉบับนี้อธิบายการใช้งาน MedCare Dashboard ตั้งแต่ขั้นตอนแรกจนถึงขั้นตอนสุดท้าย ครอบคลุมการตั้งค่าเซิร์ฟเวอร์ การเข้าสู่ระบบ การเพิ่มอุปกรณ์ การจัดการยา ตารางจ่ายยา การแจ้งเตือน ผู้ดูแล ประวัติการจ่ายยา และการตรวจสอบระบบ

วันที่จัดทำ: 17 มิถุนายน 2026  
ระบบ: MedCare Dashboard  
เว็บไซต์ใช้งานจริง: https://medcare.i-esaan.com/  
API ใช้งานจริง: https://medcare.i-esaan.com/api/v1

---

## 1. ภาพรวมของ Dashboard

MedCare Dashboard เป็นหน้าเว็บสำหรับดูแลระบบตู้จ่ายยาอัตโนมัติ ใช้สำหรับจัดการอุปกรณ์ MedCare, เพิ่มรายการยา, กำหนดช่องใส่ยา, สร้างตารางเวลาแจ้งเตือนและจ่ายยา, ตรวจสอบประวัติ, ตั้งค่าการแจ้งเตือน และดูสถานะระบบแบบรวมศูนย์

ผู้ใช้งานหลักของ Dashboard ได้แก่

- เจ้าของระบบหรือผู้ดูแลระบบ
- เจ้าหน้าที่หรือผู้ช่วยดูแลผู้ป่วย
- ผู้ดูแลที่ต้องการดูสถานะการกินยา
- ผู้ใช้งานที่ต้องการตรวจสอบประวัติและแจ้งเตือน

Dashboard เหมาะสำหรับการใช้งานผ่านคอมพิวเตอร์ แท็บเล็ต หรือเบราว์เซอร์มือถือ แต่ฟังก์ชันเพิ่มอุปกรณ์ผ่าน Bluetooth ควรใช้ผ่าน Chrome หรือ Microsoft Edge

## 2. สิ่งที่ต้องเตรียมก่อนเริ่มใช้งาน

ก่อนเริ่มใช้งาน Dashboard ควรเตรียมสิ่งต่อไปนี้ให้พร้อม

### 2.1 อุปกรณ์ที่ต้องมี

- คอมพิวเตอร์ แท็บเล็ต หรือมือถือที่เปิดเว็บเบราว์เซอร์ได้
- อินเทอร์เน็ตหรือเครือข่ายภายในที่เชื่อมต่อกับ Backend ได้
- อุปกรณ์ MedCare Dispenser หรือ ESP32 MedCare
- Wi-Fi สำหรับให้อุปกรณ์ MedCare เชื่อมต่อ
- บัญชีผู้ใช้งาน Dashboard
- รายการยาที่ต้องการจัดการ
- ข้อมูลผู้ดูแลหรือสมาชิกทีม หากต้องการเชิญให้ช่วยดูแล

### 2.2 เบราว์เซอร์ที่แนะนำ

| การใช้งาน | เบราว์เซอร์ที่แนะนำ |
| --- | --- |
| ใช้งาน Dashboard ทั่วไป | Chrome, Edge, Firefox, Safari |
| เพิ่มอุปกรณ์ผ่าน Bluetooth | Chrome หรือ Microsoft Edge |
| ตรวจสอบสถานะและดูข้อมูล | Chrome, Edge, Firefox |

หมายเหตุ: Firefox โดยทั่วไปไม่รองรับ Web Bluetooth ดังนั้นหากต้องเพิ่มอุปกรณ์ผ่าน Bluetooth ให้ใช้ Chrome หรือ Microsoft Edge แทน

### 2.3 เงื่อนไขสำหรับเพิ่มอุปกรณ์ผ่าน Bluetooth

หากต้องการเพิ่มอุปกรณ์ผ่านหน้า Add Device Wizard ต้องมีเงื่อนไขดังนี้

- เปิด Dashboard ผ่าน HTTPS หรือ localhost
- ใช้ Chrome หรือ Microsoft Edge
- เครื่องคอมพิวเตอร์ต้องมี Bluetooth
- เปิด Bluetooth ของเครื่องแล้ว
- อุปกรณ์ MedCare ต้องอยู่ในโหมดพร้อมเชื่อมต่อ
- อยู่ใกล้อุปกรณ์ในระยะที่ Bluetooth ตรวจพบได้

## 3. สิทธิ์ผู้ใช้งานและบทบาท

ระบบแบ่งสิทธิ์ผู้ใช้งานตามบทบาท เพื่อควบคุมว่าแต่ละคนสามารถดู แก้ไข หรือจัดการระบบได้มากน้อยเพียงใด

| บทบาท | ความหมาย | สิทธิ์หลัก |
| --- | --- | --- |
| Owner | เจ้าของระบบ | ใช้งานได้ครบทุกฟังก์ชัน |
| Admin | ผู้ดูแลระบบ | จัดการอุปกรณ์ ผู้ใช้ ตารางยา แจ้งเตือน และระบบได้ |
| Helper | ผู้ช่วยดูแล | เพิ่มหรือแก้ไขยา ตารางยา และสั่งงานอุปกรณ์บางอย่างได้ |
| Viewer | ผู้ดูข้อมูล | ดูข้อมูลได้เป็นหลัก แก้ไขไม่ได้ |

| ตั้งค่าการแจ้งเตือน | ได้ | จำกัด/อ่านได้ | อ่านได้ |

หากเมนูหรือปุ่มบางอย่างไม่แสดง แสดงว่าบัญชีของคุณอาจไม่มีสิทธิ์ใช้งานฟังก์ชันนั้น

## 4. การเข้าใช้งานครั้งแรก

### 4.1 เปิดเว็บไซต์ Dashboard

1. เปิดเว็บเบราว์เซอร์
2. ไปที่ https://medcare.i-esaan.com/
3. รอให้หน้าเข้าสู่ระบบแสดงขึ้นมา
4. ตรวจสอบว่าหน้าเว็บขึ้นเป็น HTTPS และไม่มีคำเตือนเรื่องความปลอดภัย

หากหน้าเว็บเปิดไม่ได้ ให้ตรวจสอบอินเทอร์เน็ตก่อน จากนั้นตรวจสอบการตั้งค่า Server/API ตามหัวข้อที่ 5

### 4.2 เลือก Server Profile ก่อนเข้าสู่ระบบ

ในหน้า Login จะมีส่วนตั้งค่า Server Profile สำหรับเลือก Backend API ที่ Dashboard จะเชื่อมต่อ

ตัวเลือกที่พบได้บ่อย ได้แก่

| Profile | ใช้เมื่อ |
| --- | --- |
| Local dev proxy | ใช้ตอนพัฒนาบนเครื่องเดียวกัน |
| Dev/Home | ใช้กับ Backend ในเครือข่ายบ้านหรือห้องทดสอบ |
| Work/Office | ใช้กับ Backend ในเครือข่ายสำนักงาน |
| Production | ใช้งานระบบจริง |
| Custom | กำหนด URL เอง |

สำหรับระบบใช้งานจริง ให้เลือก Production ซึ่งใช้ API:

```text
https://medcare.i-esaan.com/api/v1
```

### 4.3 ทดสอบการเชื่อมต่อ API

1. เลือก Profile ที่ต้องการ
2. กดทดสอบการเชื่อมต่อ หากมีปุ่มให้ทดสอบ
3. หากระบบตอบกลับสำเร็จ แปลว่า Dashboard เชื่อมต่อ Backend ได้
4. หากเชื่อมต่อไม่สำเร็จ ให้ตรวจสอบ URL, อินเทอร์เน็ต และสถานะ Backend

### 4.4 สมัครสมาชิก

หากยังไม่มีบัญชี ให้สมัครสมาชิกก่อน

1. ที่หน้า Login กด "สมัครสมาชิกใหม่"
2. กรอกชื่อ-นามสกุล
3. กรอกอีเมล
4. กรอกรหัสผ่าน
5. ยืนยันรหัสผ่านอีกครั้ง
6. กดสมัครสมาชิก
7. หากสมัครสำเร็จ ระบบจะเข้าสู่ระบบให้อัตโนมัติ หรือพากลับไปหน้า Login

ข้อแนะนำ:

- ใช้อีเมลจริงที่ติดต่อได้
- ตั้งรหัสผ่านให้เดายาก
- อย่าใช้รหัสผ่านเดียวกับระบบสำคัญอื่น

### 4.5 เข้าสู่ระบบ

1. เปิดหน้า Login
2. ตรวจสอบ Server Profile ให้ถูกต้อง
3. กรอกอีเมล
4. กรอกรหัสผ่าน
5. หากต้องการดูรหัสผ่าน ให้กดไอคอนแสดงรหัสผ่าน
6. กดเข้าสู่ระบบ
7. เมื่อสำเร็จ ระบบจะพาเข้าสู่หน้า Dashboard หลัก

### 4.6 ลืมรหัสผ่าน

หากลืมรหัสผ่าน:

1. ที่หน้า Login กด "ลืมรหัสผ่าน?"
2. กรอกอีเมลที่ใช้สมัคร
3. ทำตามขั้นตอนที่ระบบแจ้ง
4. กลับมาเข้าสู่ระบบอีกครั้งหลังตั้งรหัสผ่านใหม่

## 5. การตั้งค่า Server/API

หน้า Server/API ใช้สำหรับกำหนดว่า Dashboard จะเชื่อมต่อกับ Backend ที่ใด

### 5.1 เปิดหน้า Server/API

1. เข้าสู่ระบบ
2. ไปที่เมนู Settings
3. เลือก Server/API
4. ตรวจสอบค่า API Base URL

### 5.2 รูปแบบ API URL ที่ถูกต้อง

API URL ต้องเป็นรูปแบบใดรูปแบบหนึ่ง:

```text
/api/v1
```

หรือ

```text
https://example.com/api/v1
```

หรือ

```text
http://192.168.x.x:3001/api/v1
```

ข้อสำคัญ:

- URL ควรลงท้ายด้วย `/api/v1`
- หากเป็นระบบจริงควรใช้ HTTPS
- หากใส่ URL ผิด Dashboard อาจ Login ไม่ได้หรือโหลดข้อมูลไม่ได้

### 5.3 ทดสอบการเชื่อมต่อ

1. ใส่ API URL ที่ต้องการ
2. กด Test Connection
3. รอผลการทดสอบ
4. หากสำเร็จ ระบบจะแสดงข้อมูลบริการหรือเวอร์ชัน
5. กด Save เพื่อบันทึก

### 5.4 Reset ค่า Server

หากตั้งค่าผิดจนใช้งานไม่ได้:

1. ไปที่หน้า Server/API
2. กด Reset
3. เลือก Production หรือ URL ที่ถูกต้อง
4. ทดสอบการเชื่อมต่ออีกครั้ง
5. บันทึกค่าใหม่

## 8. การเพิ่มอุปกรณ์ MedCare

สามารถเพิ่มอุปกรณ์ได้จากเมนู Add Device หรือ Devices

### 8.1 วิธีเพิ่มอุปกรณ์ผ่าน Add Device Wizard

เหมาะสำหรับอุปกรณ์ใหม่ที่ต้องตั้งค่า Wi-Fi และ Token

1. ใช้ Chrome หรือ Microsoft Edge
2. เข้าสู่ระบบ Dashboard
3. ไปที่เมนู Add Device
4. เตรียมอุปกรณ์ MedCare ให้เปิดเครื่อง
5. เปิด Bluetooth บนคอมพิวเตอร์
6. กดเริ่มค้นหาอุปกรณ์
7. เลือกอุปกรณ์ที่ขึ้นต้นด้วยชื่อ MedCare หรือ ESP32
8. รอให้ Dashboard เชื่อมต่อกับอุปกรณ์
9. กรอกชื่ออุปกรณ์
10. ตรวจสอบ MAC Address หากระบบอ่านได้
11. กรอกชื่อ Wi-Fi
12. กรอกรหัสผ่าน Wi-Fi
13. กรอกตำแหน่งวางอุปกรณ์ เช่น ห้องนอน หรือห้องผู้ป่วย
14. ตรวจสอบ API Base URL ที่จะส่งให้อุปกรณ์
15. กดยืนยัน
16. รอให้ระบบสร้างอุปกรณ์และส่งข้อมูลไปยังอุปกรณ์ผ่าน Bluetooth
17. เมื่อสำเร็จ ระบบจะพาไปหน้า Devices
18. ตรวจสอบว่าอุปกรณ์เริ่มออนไลน์

### 8.2 สิ่งที่ระบบทำระหว่างเพิ่มอุปกรณ์

ระหว่างขั้นตอนเพิ่มอุปกรณ์ ระบบจะ:

- ค้นหาอุปกรณ์ผ่าน Bluetooth
- อ่านข้อมูลสถานะจากอุปกรณ์ หากอุปกรณ์ส่งข้อมูลกลับมา
- สร้างหรืออัปเดตอุปกรณ์ใน Backend
- สร้าง Device Token
- ส่ง Wi-Fi SSID, Wi-Fi Password, API URL และ Token ไปยังอุปกรณ์
- ให้ ESP32 ใช้ข้อมูลดังกล่าวเชื่อมต่อระบบ

### 8.3 กรณี MAC Address ซ้ำ

หากระบบพบว่า MAC Address ของอุปกรณ์มีอยู่แล้ว ระบบจะพยายามอัปเดตข้อมูลอุปกรณ์เดิม แทนการสร้างอุปกรณ์ซ้ำ

### 8.4 หาก Bluetooth ไม่พร้อมใช้งาน

หากเบราว์เซอร์ไม่รองรับ Bluetooth หรือเชื่อมต่อไม่ได้:

1. เปลี่ยนไปใช้ Chrome หรือ Microsoft Edge
2. ตรวจสอบว่าเปิดเว็บผ่าน HTTPS
3. เปิด Bluetooth ในเครื่อง
4. รีสตาร์ตอุปกรณ์ MedCare
5. ลองค้นหาใหม่

หากยังไม่ได้ สามารถเพิ่มอุปกรณ์แบบ Manual ผ่านหน้า Devices ได้ แต่ต้องนำ Device Token ไปตั้งค่าในอุปกรณ์เอง

## 9. การจัดการอุปกรณ์ในหน้า Devices

หน้า Devices ใช้สำหรับดูรายการอุปกรณ์ แก้ไขชื่อ ดูสถานะ ดูข้อมูลเชิงเทคนิค และสั่งงานอุปกรณ์

### 9.1 เปิดหน้า Devices

1. เข้าสู่ระบบ
2. ไปที่เมนู Devices
3. ระบบจะแสดงรายการอุปกรณ์ทั้งหมดที่บัญชีมีสิทธิ์เข้าถึง
4. เลือกอุปกรณ์ที่ต้องการดูรายละเอียด

### 9.2 ข้อมูลที่แสดงในหน้าอุปกรณ์

ข้อมูลที่อาจแสดง ได้แก่:

- ชื่ออุปกรณ์
- UUID
- MAC Address
- สถานะออนไลน์หรือออฟไลน์
- Firmware version
- IP Address
- Wi-Fi SSID
- Wi-Fi RSSI หรือคุณภาพสัญญาณ
- สถานะ MQTT
- สถานะ WebSocket
- Heartbeat ล่าสุด
- ข้อมูลหน่วยความจำและทรัพยากร
- ประวัติคำสั่ง
- ประวัติการจ่ายยา

### 9.3 เพิ่มอุปกรณ์แบบ Manual

ใช้เมื่อไม่ต้องการเพิ่มผ่าน Bluetooth

1. ไปที่ Devices
2. กดเพิ่มอุปกรณ์
3. กรอกชื่ออุปกรณ์
4. กดบันทึก
5. ระบบจะแสดง Device Token แบบครั้งเดียว
6. คัดลอก Token และนำไปตั้งค่าในอุปกรณ์
7. ปิดหน้าต่างหลังเก็บ Token แล้ว

ข้อควรระวัง: Device Token จะแสดงเพียงครั้งเดียว หากปิดไปแล้วจะดู Token เดิมไม่ได้ ต้องสร้าง Token ใหม่

### 9.4 แก้ไขชื่ออุปกรณ์

1. เปิดหน้า Devices
2. เลือกอุปกรณ์
3. กด Edit หรือแก้ไข
4. เปลี่ยนชื่ออุปกรณ์
5. กด Save

### 9.5 สร้าง Token ใหม่

กรณีอุปกรณ์เชื่อมต่อไม่ได้เพราะ Token สูญหายหรือรั่วไหล:

1. เปิดหน้า Devices
2. เลือกอุปกรณ์
3. เลือก Regenerate Token
4. ยืนยันการสร้าง Token ใหม่
5. คัดลอก Token ใหม่ทันที
6. นำ Token ใหม่ไปตั้งค่าในอุปกรณ์

หลังสร้าง Token ใหม่ Token เดิมจะใช้ไม่ได้

### 9.6 ลบอุปกรณ์

1. เปิดหน้า Devices
2. เลือกอุปกรณ์ที่ต้องการลบ
3. กด Delete
4. อ่านคำเตือนให้ครบ
5. ยืนยันการลบ

ควรลบอุปกรณ์เฉพาะเมื่อเลิกใช้งานจริง เพราะอาจกระทบตารางยา ประวัติ และการแจ้งเตือนที่เกี่ยวข้อง

### 9.7 ทดสอบการเชื่อมต่ออุปกรณ์

1. เปิดรายละเอียดอุปกรณ์
2. กดทดสอบการเชื่อมต่อหรือ Ping
3. ระบบจะส่งคำสั่งไปยังอุปกรณ์
4. รอผลใน Command History
5. หากสำเร็จ แสดงว่า Backend ส่งคำสั่งถึงอุปกรณ์ได้

### 9.8 สั่งงานอุปกรณ์

ขึ้นอยู่กับสิทธิ์ของผู้ใช้ อาจสามารถสั่งงานได้ เช่น:

- Reload schedule
- Test servo
- Test servo cycle
- Dispense
- Play audio
- Reboot

ข้อควรระวัง: คำสั่ง Dispense อาจทำให้ยาออกจริง ควรตรวจสอบช่องยาและผู้ป่วยก่อนสั่งงานเสมอ

## 11. การกำหนดช่องยาใน Device Slots

หน้า Device Slots ใช้สำหรับกำหนดว่ายาแต่ละตัวอยู่ในช่องใดของอุปกรณ์

### 11.1 หลักการของช่องยา

อุปกรณ์ MedCare รองรับ 7 ช่องยา โดยระบบจับคู่ดังนี้:

| ช่องยา | Servo Channel |
| --- | --- |
| Slot 1 | CH 0 |
| Slot 2 | CH 1 |
| Slot 3 | CH 2 |
| Slot 4 | CH 3 |
| Slot 5 | CH 4 |
| Slot 6 | CH 5 |
| Slot 7 | CH 6 |

### 11.2 ตั้งค่าช่องยา

1. ไปที่เมนู Device Slots
2. หากมีหลายอุปกรณ์ ให้เลือกอุปกรณ์ก่อน
3. ดูรายการ Slot 1 ถึง Slot 7
4. เลือกยาที่อยู่ในแต่ละช่อง
5. เปิดหรือปิดการใช้งานช่องนั้น
6. กด Save ของแต่ละแถว หรือ Save All เพื่อบันทึกทั้งหมด

### 11.3 ข้อควรระวัง

- ยาที่เลือกใน Slot ควรตรงกับยาที่ใส่ในอุปกรณ์จริง
- หากใส่ยาผิดช่อง ตารางจ่ายยาอาจสั่งจ่ายยาผิดตัว
- หลังเปลี่ยนยาในช่อง ควรตรวจสอบตารางยาอีกครั้ง
- ถ้าเปลี่ยนโครงสร้างช่องยา ควรทดสอบ Servo หลังตั้งค่า

## 12. การปรับเทียบ Servo Calibration

หน้า Servo Calibration ใช้สำหรับตั้งค่ามุมเปิดและปิดของ servo แต่ละช่อง ใช้ได้เฉพาะผู้ใช้สิทธิ์ Admin หรือ Owner

### 12.1 เปิดหน้า Servo Calibration

1. เข้าสู่ระบบด้วยบัญชี Admin หรือ Owner
2. ไปที่ Settings
3. เลือก Servo Calibration
4. เลือกอุปกรณ์ที่ต้องการปรับเทียบ

### 12.2 ค่าพื้นฐานของ Servo

โดยทั่วไปอุปกรณ์มี 7 servo:

| Slot | Channel |
| --- | --- |
| Slot 1 | CH 0 |
| Slot 2 | CH 1 |
| Slot 3 | CH 2 |
| Slot 4 | CH 3 |
| Slot 5 | CH 4 |
| Slot 6 | CH 5 |
| Slot 7 | CH 6 |

ค่ามุมที่ใช้บ่อย:

- มุมปิด: ประมาณ 30 องศา
- มุมเปิด: ประมาณ 140 องศา
- ช่วงมุมที่รองรับ: 0 ถึง 180 องศา

### 12.3 วิธีปรับเทียบแต่ละช่อง

1. เลือก Slot ที่ต้องการปรับ
2. ปรับค่า Closed Angle เพื่อกำหนดตำแหน่งปิด
3. กดปุ่ม Close เพื่อทดสอบตำแหน่งปิด
4. ปรับค่า Open Angle เพื่อกำหนดตำแหน่งเปิด
5. กดปุ่ม Open เพื่อทดสอบตำแหน่งเปิด
6. กด Test เพื่อทดสอบรอบเปิด-ปิด
7. หาก servo หมุนกลับด้าน ให้เปิดตัวเลือก Reverse
8. กด Save ของช่องนั้น
9. ทำซ้ำจนครบทุกช่องที่ใช้งาน

### 12.4 บันทึกทั้งหมด

หลังปรับหลายช่อง:

1. ตรวจสอบค่าทุกช่อง
2. กด Save All
3. รอระบบบันทึกค่า
4. ทดสอบอุปกรณ์อีกครั้ง

### 12.5 ตั้งค่ารอบการ Poll

อาจมีค่าระบบเพิ่มเติม เช่น:

- Command poll interval
- Schedule poll interval

ให้ใช้ค่าเริ่มต้นหากไม่แน่ใจ เพราะค่าที่สั้นเกินไปอาจเพิ่มภาระให้ระบบและอุปกรณ์

### 12.6 ข้อควรระวังด้านความปลอดภัย

- ควรถอดยาออกก่อนทดสอบ Servo
- ห้ามเอามือเข้าใกล้กลไกระหว่างทดสอบ
- ทดสอบทีละช่อง
- ตรวจสอบว่าเปิด-ปิดได้จริงก่อนนำยาใส่กลับ
- หลังปรับเทียบแล้วควรทดสอบจ่ายยาจำลองก่อนใช้งานจริง

## 15. การตั้งค่าช่องทางแจ้งเตือน

หน้า Notification Settings ใช้สำหรับตั้งค่าการแจ้งเตือนซ้ำ ช่องทางแจ้งเตือน และช่วงเวลางดแจ้งเตือน

### 15.1 เปิดหน้า Notification Settings

1. ไปที่ Settings
2. เลือก Notification Settings
3. ตรวจสอบสิทธิ์ของบัญชี หากไม่ใช่ Admin/Owner อาจแก้ไขไม่ได้

### 15.2 การแจ้งเตือนซ้ำ

ตัวเลือกหลัก ได้แก่:

| ตัวเลือก | ความหมาย |
| --- | --- |
| Escalation enabled | เปิดระบบยกระดับการแจ้งเตือน |
| Repeat enabled | เปิดการแจ้งเตือนซ้ำ |
| Repeat interval minutes | ระยะห่างระหว่างการเตือนซ้ำ |
| Max repeat count | จำนวนครั้งสูงสุดที่เตือนซ้ำ |

ตัวอย่าง:

- Repeat interval: 10 นาที
- Max repeat count: 3 ครั้ง

หมายความว่า หากผู้ป่วยยังไม่ยืนยัน ระบบจะเตือนซ้ำทุก 10 นาที ไม่เกิน 3 ครั้ง

### 15.3 Quiet Hours

Quiet Hours คือช่วงเวลาที่ต้องการลดหรือปิดการแจ้งเตือนบางประเภท

1. เปิด Quiet hours enabled
2. ตั้งเวลาเริ่มต้น เช่น 21:00
3. ตั้งเวลาสิ้นสุด เช่น 07:00
4. กด Save

ควรใช้ฟังก์ชันนี้อย่างระมัดระวัง เพราะการปิดแจ้งเตือนตอนกลางคืนอาจทำให้พลาดยาสำคัญ

### 15.4 ช่องทางแจ้งเตือน

ระบบอาจรองรับช่องทางต่อไปนี้:

- In-app notification
- Push notification
- Telegram
- Email
- LINE

In-app มักเปิดใช้งานเป็นพื้นฐาน ส่วน Telegram, Email และ LINE ต้องตั้งค่าข้อมูลเชื่อมต่อให้ครบก่อน

### 15.5 ตั้งค่า Telegram

1. เปิดช่องทาง Telegram
2. กรอก Bot Token
3. กรอก Chat ID
4. บันทึก
5. ทดสอบการแจ้งเตือน

### 15.6 ตั้งค่า Email

1. เปิดช่องทาง Email
2. กรอกอีเมลผู้รับ
3. บันทึก

## 16. การสร้างตารางจ่ายยา

หน้า Schedules ใช้สำหรับกำหนดว่าเวลาใด อุปกรณ์ใด ช่องใด และยาใดจะถูกแจ้งเตือนหรือจ่ายยา

### 16.1 เปิดหน้า Schedules

1. เข้าสู่ระบบ
2. ไปที่เมนู Schedules
3. ดูรายการตารางที่มีอยู่
4. กด Add Schedule หรือ New Schedule เพื่อสร้างรายการใหม่

### 16.2 ข้อมูลที่ต้องใช้

ก่อนสร้างตาราง ควรมีข้อมูลเหล่านี้พร้อม:

- อุปกรณ์
- ยา
- ช่องยาที่ตั้งค่าแล้ว
- เวลา
- ช่วงมื้ออาหาร
- ไฟล์เสียงแจ้งเตือน หากต้องการ

### 16.3 สร้างตารางใหม่

1. กด Add Schedule
2. เลือกอุปกรณ์
3. เลือกช่องยา
4. เลือกยาที่ต้องการให้จ่าย
5. ตั้งเวลา เช่น 08:30
6. เลือกช่วงมื้ออาหาร เช่น หลังอาหารเช้า
7. เลือกไฟล์เสียงแจ้งเตือน หากต้องการ
8. ใส่หมายเหตุถ้ามี
9. เปิด Active
10. กด Save

### 16.4 ช่วงมื้ออาหารที่ระบบมีให้

| ช่วงเวลา | เวลาแนะนำ |
| --- | --- |
| ก่อนอาหารเช้า | 06:30 |
| หลังอาหารเช้า | 08:30 |
| ก่อนอาหารกลางวัน | 11:30 |
| หลังอาหารกลางวัน | 14:30 |
| ก่อนอาหารเย็น | 17:30 |
| หลังอาหารเย็น | 20:30 |
| ก่อนนอน | 22:30 |

ตารางใน Dashboard ใช้ซ้ำทุกวันของสัปดาห์ หากต้องการหยุดใช้ชั่วคราว ให้ปิด Active ของตารางนั้น

### 16.5 เลือกหลายยาในตารางเดียว

บางกรณีสามารถเลือกยาได้หลายรายการในตารางเดียว สูงสุดตามที่ระบบกำหนด เช่น 7 รายการ แต่ต้องตรวจสอบให้แน่ใจว่าช่องยาและอุปกรณ์รองรับการทำงานนั้น

### 16.6 แก้ไขตารางยา

1. เปิดหน้า Schedules
2. เลือกรายการที่ต้องการแก้ไข
3. แก้ไขเวลา ช่องยา เสียง หรือสถานะ Active
4. กด Save

ในบางมุมมองสามารถแก้ไขข้อมูลบางรายการจากตารางโดยตรง เช่น เวลา ช่องยา ช่วงมื้ออาหาร ไฟล์เสียง และสถานะเปิดใช้งาน

### 16.7 ปิดใช้งานตารางยา

หากไม่ต้องการให้ตารางทำงานชั่วคราว:

1. เปิดหน้า Schedules
2. หาตารางที่ต้องการ
3. ปิด Active
4. กด Save

การปิด Active เหมาะกว่าการลบทิ้งเมื่อยังอาจกลับมาใช้ตารางเดิมในอนาคต

### 16.8 ตรวจสอบหลังสร้างตาราง

หลังสร้างหรือแก้ไขตาราง ควรตรวจสอบ:

- อุปกรณ์ถูกต้อง
- ช่องยาถูกต้อง
- ยาถูกต้อง
- เวลาไม่ซ้ำหรือใกล้กันเกินไป
- เสียงแจ้งเตือนถูกต้อง
- สถานะ Active เปิดอยู่
- อุปกรณ์ออนไลน์และรับตารางใหม่แล้ว

### 25.3 หลังจบวัน

## 27. การแก้ปัญหาเบื้องต้น

### 27.1 เข้าเว็บไม่ได้

สาเหตุที่เป็นไปได้:

- อินเทอร์เน็ตมีปัญหา
- เว็บไซต์ปลายทางไม่ตอบสนอง
- DNS หรือ SSL มีปัญหาชั่วคราว
- เบราว์เซอร์แคชข้อมูลเก่า

วิธีแก้:

1. ลองรีเฟรชหน้า
2. ลองเปิดด้วย Chrome และ Firefox เปรียบเทียบกัน
3. ตรวจสอบอินเทอร์เน็ต
4. ล้าง Cache หรือเปิด Incognito
5. ตรวจสอบว่า URL เป็น `https://medcare.i-esaan.com/`

### 27.2 Login ไม่ได้

สาเหตุที่เป็นไปได้:

- อีเมลหรือรหัสผ่านผิด
- เลือก Server Profile ผิด
- API URL ไม่ถูกต้อง
- Backend ไม่ตอบสนอง

วิธีแก้:

1. ตรวจสอบอีเมลและรหัสผ่าน
2. ตั้ง Server Profile เป็น Production
3. ตรวจสอบ API URL
4. กด Test Connection
5. ใช้ Forgot Password หากจำรหัสผ่านไม่ได้

### 27.3 เพิ่มอุปกรณ์ผ่าน Bluetooth ไม่ได้

สาเหตุที่เป็นไปได้:

- ใช้ Firefox หรือเบราว์เซอร์ที่ไม่รองรับ Web Bluetooth
- ไม่ได้เปิด Bluetooth
- เปิดเว็บผ่าน HTTP แทน HTTPS
- อุปกรณ์ไม่อยู่ในโหมดพร้อมเชื่อมต่อ
- ระยะห่างเกินไป

วิธีแก้:

1. ใช้ Chrome หรือ Microsoft Edge
2. เปิด Bluetooth
3. เปิดเว็บผ่าน HTTPS
4. รีสตาร์ตอุปกรณ์
5. วางอุปกรณ์ใกล้คอมพิวเตอร์
6. ลองค้นหาใหม่

### 27.4 อุปกรณ์ออฟไลน์

สาเหตุที่เป็นไปได้:

- อุปกรณ์ไม่ได้เปิด
- Wi-Fi ใช้งานไม่ได้
- ใส่ Wi-Fi Password ผิด
- Device Token ผิดหรือหมดอายุ
- Backend หรือ MQTT ติดต่อไม่ได้

วิธีแก้:

1. ตรวจสอบไฟเลี้ยงอุปกรณ์
2. ตรวจสอบ Wi-Fi
3. เปิดหน้า Devices ดู Heartbeat ล่าสุด
4. เปิด Diagnostics ดู MQTT และ Backend
5. ส่งคำสั่ง reload_schedule
6. หาก Token ผิด ให้ Regenerate Token และตั้งค่าอุปกรณ์ใหม่

### 27.5 ตารางยาไม่ทำงาน

สาเหตุที่เป็นไปได้:

- ตารางไม่ได้ Active
- อุปกรณ์ออฟไลน์
- ช่องยาไม่ได้เปิดใช้งาน
- ยาไม่ได้ผูกกับช่อง
- เวลาในอุปกรณ์หรือ Backend ไม่ตรง
- อุปกรณ์ยังไม่ได้โหลดตารางล่าสุด

วิธีแก้:

1. เปิด Schedules ตรวจสอบ Active
2. เปิด Device Slots ตรวจสอบช่องยา
3. เปิด Devices ตรวจสอบสถานะออนไลน์
4. ไปที่ Diagnostics แล้วสั่ง reload_schedule
5. ตรวจสอบ Dose Events ว่ามีรอบยาถูกสร้างหรือไม่
6. ตรวจสอบ History หลังถึงเวลา

### 27.6 ยาออกผิดช่อง

สาเหตุที่เป็นไปได้:

- ผูกยาใน Device Slots ผิด
- เลือก Slot ใน Schedule ผิด
- Servo Channel สลับกับช่องจริง
- ปรับ Servo ผิดตำแหน่ง

วิธีแก้:

1. หยุดใช้งานตารางชั่วคราว
2. ตรวจสอบ Device Slots
3. ตรวจสอบ Schedules
4. ทดสอบ Servo ทีละช่อง
5. ปรับ Servo Calibration
6. ทดลองจ่ายยาโดยไม่มีผู้ป่วยใช้งานจริงก่อน

### 27.7 ไม่ได้รับแจ้งเตือน

สาเหตุที่เป็นไปได้:

- ช่องทางแจ้งเตือนยังไม่เปิด
- ตั้งค่า Telegram, Email หรือ LINE ไม่ครบ
- Repeat notification ปิดอยู่
- Quiet Hours ครอบคลุมเวลานั้น
- ผู้ดูแลไม่มีสิทธิ์หรือไม่ได้อยู่ในทีม

วิธีแก้:

1. เปิด Notification Settings
2. ตรวจสอบช่องทางแจ้งเตือน
3. ตรวจสอบข้อมูล Token หรือผู้รับ
4. ตรวจสอบ Quiet Hours
5. ดู Notifications ในระบบก่อน
6. ตรวจสอบ Caregivers ว่าผู้ดูแล Active อยู่

### 27.8 เสียงไม่เล่น

สาเหตุที่เป็นไปได้:

- ไม่ได้เปิด Alert audio
- ไฟล์เสียงไม่ Active
- ตารางยาไม่ได้เลือกไฟล์เสียง
- อุปกรณ์ยังไม่ได้รับไฟล์เสียงหรือ config ล่าสุด
- ลำโพงหรือโมดูลเสียงมีปัญหา

วิธีแก้:

1. เปิด Device Alert Settings
2. ตรวจสอบ Audio Files ว่าไฟล์ Active
3. ตรวจสอบ Schedule ว่าเลือกเสียงแล้ว
4. ใช้ Diagnostics ทดสอบ play_audio
5. ตรวจสอบฮาร์ดแวร์เสียงที่อุปกรณ์

### 27.9 ไม่มีปุ่มแก้ไขหรือลบ

สาเหตุที่เป็นไปได้:

- บัญชีเป็น Viewer
- บัญชีไม่มีสิทธิ์ Admin
- ฟังก์ชันนั้นจำกัดเฉพาะ Owner/Admin

วิธีแก้:

1. ตรวจสอบบทบาทใน Profile
2. ติดต่อ Owner หรือ Admin
3. ขอปรับสิทธิ์ในหน้า Caregivers หากเหมาะสม

### 27.10 Export CSV ไม่ได้

สาเหตุที่เป็นไปได้:

- เบราว์เซอร์บล็อกการดาวน์โหลด
- ไม่มีข้อมูลตามตัวกรอง
- Session หมดอายุ

วิธีแก้:

1. รีเฟรชหน้า
2. เข้าสู่ระบบใหม่
3. เปลี่ยนช่วงวันที่
4. อนุญาตการดาวน์โหลดจากเว็บไซต์
5. ลองใช้ Chrome หรือ Edge

## 28. แนวทางความปลอดภัย

### 28.1 ความปลอดภัยของบัญชี

- ใช้รหัสผ่านที่เดายาก
- อย่าแชร์บัญชีผู้ใช้ร่วมกัน
- ออกจากระบบเมื่อใช้เครื่องสาธารณะ
- จำกัดสิทธิ์ผู้ดูแลตามหน้าที่จริง

### 28.2 ความปลอดภัยของ Device Token

- เก็บ Device Token เป็นความลับ
- อย่าส่ง Token ผ่านช่องทางที่ไม่ปลอดภัย
- หากสงสัยว่า Token รั่วไหล ให้ Regenerate Token ทันที
- หลังสร้าง Token ใหม่ ต้องตั้งค่าอุปกรณ์ใหม่ด้วย Token ใหม่

### 28.3 ความปลอดภัยของยา

- ตรวจสอบยาในช่องจริงให้ตรงกับ Device Slots
- ตรวจสอบตารางยาทุกครั้งหลังแก้ไข
- ทดสอบอุปกรณ์หลังปรับ Servo
- ไม่ควรสั่ง Dispense หากไม่แน่ใจว่าอยู่ในสถานการณ์ปลอดภัย
- เก็บยาให้พ้นมือเด็กและผู้ที่ไม่เกี่ยวข้อง

### 28.4 ความปลอดภัยของการแจ้งเตือน

- ตั้งผู้รับแจ้งเตือนให้ถูกต้อง
- ทดสอบ Telegram, Email หรือ LINE หลังตั้งค่า
- อย่าปิดแจ้งเตือนซ้ำหากผู้ป่วยมีความเสี่ยงพลาดยา
- ใช้ Quiet Hours อย่างระมัดระวัง
