# คู่มือการใช้งานตัวเครื่อง MedCare Dispenser

เอกสารฉบับนี้เป็นคู่มือการใช้งานตัวเครื่อง MedCare Dispenser ตั้งแต่ขั้นตอนแรกจนถึงขั้นตอนสุดท้าย ครอบคลุมการเตรียมเครื่อง การเปิดเครื่อง การจับคู่กับระบบ การใส่ยา การยืนยันรับยา การดูแลรักษา และการแก้ปัญหาเบื้องต้น

วันที่จัดทำ: 17 มิถุนายน 2026  
ระบบ: MedCare Dispenser  
ประเภทอุปกรณ์: ตู้จ่ายยาอัตโนมัติควบคุมด้วย ESP32  
เว็บไซต์ Dashboard: https://medcare.i-esaan.com/  
API ใช้งานจริง: https://medcare.i-esaan.com/api/v1

---

## 1. ภาพรวมของตัวเครื่อง

MedCare Dispenser เป็นตัวเครื่องสำหรับช่วยเตือนและจ่ายยาตามเวลาที่กำหนด เครื่องเชื่อมต่อกับระบบ MedCare ผ่าน Wi-Fi และรับตารางยาจาก Dashboard หรือ Mobile App

เมื่อถึงเวลารับยา เครื่องจะเข้าสู่สถานะเตือน ผู้ใช้ต้องกดปุ่มยืนยันรับยา จากนั้นเครื่องจึงสั่ง servo ให้เปิดช่องยาและปล่อยยาลงสู่ถ้วยรับยา หากไม่มีการกดปุ่มภายในเวลาที่กำหนด ระบบจะบันทึกว่าเป็นรอบยาที่พลาด

ลำดับการทำงานหลักของเครื่องคือ:

1. เปิดเครื่อง
2. เชื่อมต่อ Wi-Fi
3. เชื่อมต่อ Backend/Server
4. โหลดค่าตั้งค่าและตารางยา
5. รอถึงเวลายา
6. เปิดไฟ/เสียงเตือน
7. รอผู้ใช้กดปุ่มยืนยัน
8. จ่ายยาออกจากช่องที่กำหนด
9. บันทึกผลกลับไปยังระบบ
10. กลับสู่สถานะพร้อมใช้งาน

## 2. ข้อควรระวังก่อนใช้งาน

### 2.1 ข้อควรระวังด้านยา

- ตรวจสอบชนิดยา ปริมาณยา และช่องยาให้ถูกต้องทุกครั้งก่อนใช้งานจริง
- ห้ามใส่ยาผิดช่อง เพราะเครื่องจะจ่ายยาตามเลขช่องที่ตั้งไว้ในระบบ
- ไม่ควรใช้เครื่องนี้แทนการดูแลของบุคลากรทางการแพทย์ในกรณียาสำคัญหรือยาฉุกเฉิน
- หากผู้ป่วยมีความเสี่ยงสูง ควรมีผู้ดูแลตรวจสอบการกินยาร่วมด้วย
- หากยาเปลี่ยนสี แตก เปียก ชื้น หรือหมดอายุ ให้หยุดใช้และเปลี่ยนยาใหม่
- ยาที่ต้องควบคุมอุณหภูมิ ความชื้น หรือแสงเป็นพิเศษ ควรตรวจสอบกับเภสัชกรก่อนใส่ในเครื่อง

### 2.2 ข้อควรระวังด้านไฟฟ้า

- วางเครื่องในที่แห้ง หลีกเลี่ยงน้ำและความชื้น
- ใช้อะแดปเตอร์และแหล่งจ่ายไฟที่จัดไว้สำหรับเครื่องเท่านั้น
- ห้ามดึงสายไฟขณะเครื่องกำลังจ่ายยา
- หากเครื่องมีกลิ่นไหม้ ร้อนผิดปกติ หรือมีเสียงไฟฟ้าผิดปกติ ให้ปิดเครื่องและถอดปลั๊กทันที
- ห้ามเปิดฝาครอบส่วนวงจรหากไม่ใช่ผู้ดูแลหรือช่างเทคนิค

### 2.3 ข้อควรระวังด้านกลไก

- ห้ามสอดนิ้วหรือสิ่งของเข้าไปในช่องกลไกขณะเครื่องทำงาน
- ควรถอดยาออกก่อนทดสอบ servo หรือปรับเทียบช่องยา
- หากช่องยาติดขัด ให้ปิดเครื่องก่อนตรวจสอบ
- ห้ามฝืนหมุน servo ด้วยมือ เพราะอาจทำให้เฟืองเสียหาย

## 3. ส่วนประกอบของตัวเครื่อง

ตัวเครื่อง MedCare ประกอบด้วยส่วนสำคัญต่อไปนี้

| ส่วนประกอบ | หน้าที่ |
| --- | --- |
| ESP32 | ควบคุมการทำงานหลัก เชื่อมต่อ Wi-Fi และสื่อสารกับระบบ |
| PCA9685 Servo Driver | ควบคุม servo หลายช่อง |
| Servo 7 ตัว | เปิดและปิดช่องยาแต่ละช่อง |
| ช่องยา 1-7 | เก็บยาแยกตามรายการยา |
| ปุ่มรับยา | ให้ผู้ใช้กดยืนยันเมื่อถึงเวลารับยา |
| ไฟปุ่มรับยา | แสดงสถานะว่ามีรอบยารอการยืนยัน |
| Buzzer | ส่งเสียงเตือนเมื่อถึงเวลารับยา |
| OLED Display | แสดงสถานะเครื่อง เช่น Wi-Fi, IP, รอบยาถัดไป และ Pairing |
| Pairing LED | แสดงสถานะการจับคู่หรือตั้งค่า Wi-Fi |
| ถ้วยรับยา | รองรับยาที่ถูกปล่อยลงมา |
| microSD Card | เก็บไฟล์เสียงแจ้งเตือน หากใช้งานเสียงจาก SD |
| Speaker/DAC | ใช้เล่นเสียงแจ้งเตือน หากเปิดใช้งาน audio |
| Power Supply | จ่ายไฟให้ ESP32 และ servo |

## 4. แผนผังช่องยา

ตัวเครื่องรองรับ 7 ช่องยา โดยระบบจับคู่ช่องยาเข้ากับ servo channel ดังนี้

| ช่องยาในเครื่อง | Servo Channel | ชื่อที่พบในระบบ |
| --- | --- | --- |
| ช่อง 1 | CH 0 | Slot 1 |
| ช่อง 2 | CH 1 | Slot 2 |
| ช่อง 3 | CH 2 | Slot 3 |
| ช่อง 4 | CH 3 | Slot 4 |
| ช่อง 5 | CH 4 | Slot 5 |
| ช่อง 6 | CH 5 | Slot 6 |
| ช่อง 7 | CH 6 | Slot 7 |

ข้อสำคัญ: ผู้ใช้ทั่วไปควรจำตามเลขช่อง 1-7 บนตัวเครื่อง ส่วน CH 0-6 เป็นเลขเชิงเทคนิคที่ใช้ใน firmware และการปรับเทียบ

## 5. สถานะที่ผู้ใช้ควรรู้

### 5.1 สถานะปกติ

เครื่องพร้อมใช้งานเมื่อ:

- OLED แสดงว่า Wi-Fi connected
- มี IP Address แสดงบนจอ
- Server ไม่ offline
- Dashboard หรือ Mobile App เห็นอุปกรณ์เป็น online
- ไม่มี Pending dose ค้างอยู่

### 5.2 สถานะ Pairing Mode

Pairing Mode คือสถานะที่เครื่องเปิด Bluetooth Low Energy เพื่อรับค่า Wi-Fi, API URL และ Device Token จาก Dashboard หรือ Mobile App

เครื่องจะเข้า Pairing Mode เมื่อ:

- ยังไม่มีค่า Wi-Fi ในเครื่อง
- เชื่อมต่อ Wi-Fi ไม่สำเร็จ
- มีการล้างค่า Wi-Fi หรือ reset config
- ผู้ดูแลสั่งตั้งค่าเครื่องใหม่จากเครื่องมือที่รองรับ

เมื่อเข้า Pairing Mode:

- OLED จะแสดง `PAIRING MODE`
- ชื่ออุปกรณ์จะขึ้นในรูปแบบ `MedCare-XXXXXX`
- Pairing LED จะกะพริบ
- เครื่องจะเปิดหน้าต่าง Pairing ประมาณ 5 นาที
- หากยังจับคู่ไม่สำเร็จ เครื่องจะลองเชื่อมต่อ Wi-Fi ใหม่ แล้วเปิด Pairing รอบถัดไปหากยังล้มเหลว

### 5.3 สถานะเตือนรับยา

เมื่อถึงเวลารับยา:

- ไฟปุ่มรับยาจะติดหรือกะพริบ
- Buzzer จะดัง หากเปิดใช้งาน
- เสียงแจ้งเตือนจะเล่น หากเปิดใช้งาน audio
- OLED จะแสดงจำนวน Pending dose
- เครื่องจะรอให้ผู้ใช้กดปุ่มรับยา

### 5.4 สถานะพลาดยา

หากไม่กดปุ่มรับยาภายในเวลาที่กำหนด:

- เครื่องจะหยุดสถานะเตือน
- ระบบจะบันทึกเป็น missed หรือ skipped ด้วยเหตุผล button timeout
- Dashboard/Mobile App จะแสดงรายการพลาดยา
- ผู้ดูแลอาจได้รับแจ้งเตือนตามค่าที่ตั้งไว้

ค่าเริ่มต้นของเวลารอยืนยันคือ 10 นาที แต่ผู้ดูแลสามารถตั้งค่าได้ตั้งแต่ 1-60 นาทีใน Device Alert Settings

## 6. ความหมายของปุ่ม ไฟ เสียง และหน้าจอ

### 6.1 ปุ่มรับยา

ปุ่มรับยาเป็นปุ่มหลักบนตัวเครื่อง

| การกด | ผลลัพธ์ |
| --- | --- |
| กดสั้นขณะมีรอบยารออยู่ | ยืนยันรับยาและสั่งจ่ายยา |
| กดสั้นขณะไม่มีรอบยารออยู่ | เครื่องจะไม่จ่ายยา อาจกะพริบไฟเพื่อบอกว่าไม่มีรอบยาค้าง |
| กดค้างประมาณ 2 วินาทีขณะไม่มีรอบยารออยู่ | ทดสอบเสียง Buzzer |

หมายเหตุ: ใน firmware ปัจจุบัน ปุ่มรับยาไม่ได้ใช้เป็นวิธีหลักในการเข้า Pairing Mode การเข้า Pairing Mode เกิดจาก Wi-Fi หาย ตั้งค่าไม่ครบ หรือคำสั่ง reset/clear config

### 6.2 ไฟปุ่มรับยา

ไฟปุ่มรับยาใช้บอกสถานะรอบยา

| สถานะไฟ | ความหมาย |
| --- | --- |
| ดับ | ไม่มีรอบยารออยู่ |
| ติดหรือกะพริบ | มีรอบยารอการยืนยัน |
| กะพริบสั้นหลังยืนยัน | รับคำสั่งยืนยันแล้ว |

### 6.3 Buzzer

Buzzer ใช้แจ้งเตือนเมื่อถึงเวลารับยา

- หากเปิด Alert Buzzer เครื่องจะส่งเสียงเตือนเมื่อถึงเวลายา
- เสียงจะหยุดเมื่อผู้ใช้กดปุ่มรับยา หรือครบเวลารอยืนยัน
- หากกดปุ่มค้างประมาณ 2 วินาทีตอนไม่มีรอบยาค้าง เครื่องจะทดสอบ Buzzer

### 6.4 เสียง Audio

หากเปิด Alert Audio และมีไฟล์เสียงพร้อมใช้งาน เครื่องสามารถเล่นเสียงแจ้งเตือนจากไฟล์เสียงได้

ข้อควรรู้:

- Buzzer เปิดเป็นค่าเริ่มต้น
- Audio อาจปิดเป็นค่าเริ่มต้น ต้องเปิดจาก Dashboard หรือ Mobile App
- ไฟล์เสียงควรถูก sync ลงเครื่องก่อนใช้งานจริง
- หากไม่มีเสียง ให้ตรวจสอบ Audio Files, SD Card, Speaker และ Device Alert Settings

### 6.5 OLED Display

OLED แสดงสถานะสำคัญของเครื่อง เช่น:

- `MedCare booting` - เครื่องกำลังเริ่มทำงาน
- `PAIRING MODE` - เครื่องกำลังรอจับคู่
- `Device: MedCare-XXXXXX` - ชื่อเครื่องสำหรับค้นหาผ่าน Bluetooth
- `WiFi: connected` - เชื่อมต่อ Wi-Fi แล้ว
- `WiFi: offline` - ยังไม่เชื่อมต่อ Wi-Fi
- `IP:` - IP Address ของเครื่องในเครือข่าย
- `Server:` - สถานะการเชื่อมต่อระบบ
- `Next dose:` - รอบยาถัดไป
- `Pending:` - จำนวนรอบยาที่กำลังรอการกดปุ่ม
- `MQTT:OK` หรือ `MQTT:NO` - สถานะคำสั่งแบบ realtime ผ่าน MQTT
- `WS:OK` หรือ `WS:NO` - สถานะ WebSocket ถ้าใช้ profile ที่รองรับ
- `BLE: MedCare-XXXXXX` - ชื่อ BLE ของเครื่อง

## 7. การเตรียมสถานที่ติดตั้ง

ก่อนเปิดใช้งานเครื่อง ควรเลือกสถานที่ติดตั้งให้เหมาะสม

### 7.1 จุดวางเครื่องที่เหมาะสม

- วางบนพื้นเรียบ แข็งแรง และไม่เอียง
- อยู่ใกล้ปลั๊กไฟ
- อยู่ในพื้นที่ที่สัญญาณ Wi-Fi 2.4 GHz เสถียร
- อยู่ในตำแหน่งที่ผู้ป่วยมองเห็นและได้ยินเสียงเตือนได้
- มีพื้นที่ให้หยิบถ้วยรับยาได้สะดวก
- ไม่โดนแดดจัด น้ำ ความชื้น หรือไอความร้อน
- ไม่อยู่ในจุดที่เด็กหรือสัตว์เลี้ยงเข้าถึงง่าย

### 7.2 สิ่งที่ควรหลีกเลี่ยง

- วางใกล้อ่างล้างมือ ห้องน้ำ หรือจุดเปียกน้ำ
- วางบนโต๊ะที่สั่นหรือโยกง่าย
- วางใกล้ไมโครเวฟหรืออุปกรณ์ที่รบกวน Wi-Fi
- วางในตู้ปิดทึบจนเสียงเตือนเบาเกินไป
- วางในที่ร้อนจัดหรือแดดส่องนาน

- [ ] ตัวเครื่องตั้งบนพื้นเรียบ
- [ ] ไม่มีสิ่งของค้างในทางปล่อยยา
- [ ] ถ้วยรับยาวางตรงตำแหน่ง
- [ ] ช่องยาไม่มีเศษยา เศษกล่อง หรือสิ่งแปลกปลอม

- [ ] อะแดปเตอร์หรือ power supply อยู่ในสภาพปกติ
- [ ] มือแห้งก่อนจับปลั๊กและตัวเครื่อง
- [ ] มีบัญชี Dashboard หรือ Mobile App พร้อมใช้งาน
- [ ] มี Wi-Fi SSID และรหัสผ่าน
- [ ] มีข้อมูลยาและตารางยาที่ต้องการตั้งค่า

## 9. การเปิดเครื่อง

### 9.1 เปิดเครื่องครั้งแรก

1. ตรวจสอบว่าถ้วยรับยาอยู่ในตำแหน่ง
2. ตรวจสอบว่าไม่มีมือหรือสิ่งของอยู่ในช่องกลไก
3. เสียบปลั๊กหรือเปิดสวิตช์หลักของเครื่อง
4. รอให้ OLED แสดงข้อความเริ่มต้น เช่น `MedCare booting`
5. รอประมาณ 30-60 วินาทีให้เครื่องเริ่มระบบ
6. สังเกตสถานะบน OLED

### 9.2 ผลลัพธ์ที่อาจพบหลังเปิดเครื่อง

| สิ่งที่เห็น | ความหมาย | สิ่งที่ควรทำ |
| --- | --- | --- |
| OLED แสดง Wi-Fi connected | เครื่องเชื่อมต่อ Wi-Fi ได้ | ไปขั้นตอนตรวจสอบ Dashboard |
| OLED แสดง PAIRING MODE | เครื่องรอรับค่า Wi-Fi/API | ไปขั้นตอนจับคู่เครื่อง |
| OLED ไม่ติด | เครื่องอาจไม่มีไฟหรือจอไม่ทำงาน | ตรวจสอบปลั๊กและ power supply |
| Pairing LED กะพริบ | เครื่องเปิด Pairing Mode | จับคู่ภายใน 5 นาที |
| Buzzer ดัง | อาจมีรอบยารออยู่หรือกำลังทดสอบ | ตรวจสอบไฟปุ่มและ Dashboard |

## 13. การปรับเทียบช่องยาและ Servo ก่อนใช้งานจริง

การปรับเทียบใช้กำหนดมุมปิดและมุมเปิดของแต่ละช่อง เพื่อให้เครื่องปล่อยยาได้ถูกต้อง

### 13.1 ค่าพื้นฐาน

ค่าเริ่มต้นที่ใช้ใน firmware คือ:

| ค่า | ความหมาย | ค่าเริ่มต้น |
| --- | --- | --- |
| Closed / Lock | ตำแหน่งปิดช่องยา | 30 องศา |
| Open / Release | ตำแหน่งเปิดปล่อยยา | 140 องศา |

ค่าจริงอาจต่างกันตามตำแหน่งติดตั้ง servo, ความยาวแขน servo และกลไกของช่องยา

### 13.2 ข้อควรระวังก่อนปรับเทียบ

- ถอดยาออกก่อนปรับเทียบ
- อย่าให้มืออยู่ใกล้กลไก
- ทดสอบทีละช่อง
- หาก servo มีเสียงคราง สั่น หรือติด ให้หยุดทันที
- ห้ามตั้งมุมจน servo ฝืนชนจุดหยุดทางกล

### 13.3 ขั้นตอนปรับเทียบผ่าน Dashboard

1. เปิด Dashboard
2. ไปที่ Settings
3. เลือก Servo Calibration
4. เลือกอุปกรณ์
5. เริ่มจาก Slot 1
6. กดไปตำแหน่ง Closed/Min
7. ตรวจสอบว่าแขน servo ปิดช่องสนิท
8. ปรับค่ามุมปิดจนปลอดภัย
9. กดไปตำแหน่ง Open/Max
10. ตรวจสอบว่าช่องเปิดพอให้ยาหรือกล่องยาหล่นได้
11. ปรับค่ามุมเปิดจนปล่อยได้พอดี
12. กด Test เพื่อทดสอบรอบปิด - เปิด - ปิด
13. หากการเคลื่อนที่ถูกต้อง ให้กด Save
14. ทำซ้ำ Slot 1-7
15. เมื่อครบแล้วกด Save All หากมี

### 13.4 เกณฑ์ผ่านการปรับเทียบ

แต่ละช่องควรผ่านเกณฑ์ดังนี้

- ตำแหน่งปิดล็อกยาได้จริง
- ตำแหน่งเปิดปล่อยยาได้หนึ่งรอบอย่างสม่ำเสมอ
- หลังปล่อยยา servo กลับมาปิด
- ไม่มีเสียงฝืนหรือชนกลไก
- ESP32 ไม่รีสตาร์ตระหว่าง servo ทำงาน
- ทดลองซ้ำหลายครั้งแล้วไม่ติดขัด

## 14. การตั้งค่าตารางยาให้เครื่อง

ตารางยาถูกตั้งค่าจาก Dashboard หรือ Mobile App เครื่องจะดึงตารางมาเก็บไว้ในเครื่อง

- อุปกรณ์ออนไลน์
- รายการยาในระบบ
- Device Slots ตั้งค่าตรงกับช่องจริง
- Servo Calibration ผ่านแล้ว
- Device Alert Settings ตั้งค่าแล้ว
- Notification Settings ตั้งค่าแล้ว หากต้องการแจ้งเตือนผู้ดูแล

### 14.2 ขั้นตอนตั้งตาราง

1. เปิด Dashboard หรือ Mobile App
2. ไปที่ Schedules
3. สร้างตารางใหม่
4. เลือกอุปกรณ์
5. เลือกช่องยา
6. เลือกยา
7. ตั้งเวลา
8. เลือกช่วงมื้ออาหาร
9. เลือกไฟล์เสียงแจ้งเตือน หากต้องการ
10. เปิด Active
11. บันทึก
12. สั่ง reload_schedule หรือรอเครื่องดึงตารางรอบถัดไป
13. ตรวจสอบว่า Dashboard แสดงตารางถูกต้อง

### 14.3 เครื่องเก็บตารางสำรองอย่างไร

เครื่องสามารถ cache ตารางยาที่ active ไว้ในหน่วยความจำภายในได้ หาก server ใช้งานไม่ได้ชั่วคราว เครื่องยังสามารถใช้ตารางล่าสุดที่เคยโหลดไว้ได้

ข้อควรระวัง:

- หากเปลี่ยนตารางแล้วเครื่องยังไม่ได้ reload_schedule เครื่องอาจยังใช้ตารางเก่า
- หากเครื่อง offline นานเกินไป การแจ้งเตือนในระบบอาจล่าช้า
- ควรตรวจสอบสถานะ online หลังแก้ไขตารางสำคัญ

### 15.4 หากไม่ได้กดปุ่มทันเวลา

## 21. การแก้ปัญหาเบื้องต้น

### 21.1 เปิดเครื่องแล้ว OLED ไม่ติด

สาเหตุที่เป็นไปได้:

- ไม่มีไฟเข้าเครื่อง
- อะแดปเตอร์หลวม
- สวิตช์ปิดอยู่
- จอ OLED หรือสายจอมีปัญหา

วิธีแก้:

1. ตรวจสอบปลั๊กไฟ
2. ตรวจสอบอะแดปเตอร์
3. ตรวจสอบสวิตช์
4. ลองปิดเปิดเครื่อง
5. หากยังไม่ติด ให้ติดต่อผู้ดูแลหรือช่างเทคนิค

### 21.2 เครื่องเข้า Pairing Mode ตลอด

สาเหตุที่เป็นไปได้:

- ยังไม่ได้ตั้งค่า Wi-Fi
- Wi-Fi Password ผิด
- Wi-Fi อยู่ไกลเกินไป
- เครื่องจำค่า Wi-Fi เก่า
- Router ปิดหรือสัญญาณไม่เสถียร

วิธีแก้:

1. ตรวจสอบ Wi-Fi SSID และ Password
2. จับคู่ใหม่ผ่าน Dashboard หรือ Mobile App
3. วางเครื่องใกล้ Router มากขึ้น
4. ตรวจสอบว่า Wi-Fi เป็น 2.4 GHz
5. หากยังไม่ได้ ให้ล้างค่า Wi-Fi และตั้งค่าใหม่

### 21.3 Dashboard เห็นเครื่อง offline

สาเหตุที่เป็นไปได้:

- เครื่องไม่ได้เชื่อมต่อ Wi-Fi
- API URL ผิด
- Device Token ผิดหรือหมดอายุ
- Backend ไม่ตอบสนอง
- MQTT/WebSocket ไม่เชื่อมต่อ แต่ HTTP fallback อาจยังทำงาน

วิธีแก้:

1. ดู OLED ว่า Wi-Fi connected หรือไม่
2. ดู IP Address บน OLED
3. เปิด Dashboard หน้า Devices
4. ตรวจสอบ Last seen
5. ลองสั่ง reload_schedule
6. หาก Token ผิด ให้ผู้ดูแล Regenerate Token และตั้งค่าเครื่องใหม่
7. หากย้ายเครือข่าย ให้จับคู่ Wi-Fi ใหม่

### 21.4 ถึงเวลายาแล้วเครื่องไม่เตือน

สาเหตุที่เป็นไปได้:

- ตารางยาไม่ Active
- เครื่องยังไม่ได้โหลดตารางล่าสุด
- เวลาในระบบไม่ตรง
- Device Slots ไม่ได้เปิดใช้งาน
- Buzzer/Audio ถูกปิด
- เครื่อง offline และไม่มีตาราง cache ที่ถูกต้อง

วิธีแก้:

1. ตรวจสอบ Schedules
2. ตรวจสอบ Device Slots
3. ตรวจสอบ Device Alert Settings
4. สั่ง reload_schedule
5. ตรวจสอบ Dose Events
6. ตรวจสอบว่าเครื่อง online

### 21.5 กดปุ่มแล้วไม่มียาออก

สาเหตุที่เป็นไปได้:

- ไม่มีรอบยารออยู่
- ช่องยาว่าง
- servo ไม่ขยับ
- กลไกติดขัด
- ช่องยาไม่ได้เปิดใช้งานในระบบ
- ตารางชี้ช่องผิด

วิธีแก้:

1. ตรวจสอบว่าไฟปุ่มกำลังกะพริบหรือไม่
2. ตรวจสอบ Dose Events ว่ามีรอบ Alerting หรือ Scheduled หรือไม่
3. ตรวจสอบยาจริงในช่อง
4. ตรวจสอบ Device Slots
5. ทดสอบ Servo Calibration โดยไม่ใส่ยา
6. หากกลไกติด ให้ปิดเครื่องก่อนตรวจสอบ

### 21.6 ยาออกผิดช่อง

สาเหตุที่เป็นไปได้:

- ใส่ยาผิดช่อง
- ตั้งค่า Device Slots ผิด
- Schedule เลือก Slot ผิด
- Servo Channel สลับกับช่องจริง

วิธีแก้:

1. หยุดใช้งานตารางชั่วคราว
2. นำยาออกเพื่อตรวจสอบ
3. เทียบช่องจริงกับ Device Slots
4. ตรวจสอบ Schedules
5. ทดสอบ servo ทีละช่อง
6. แก้ไข mapping ให้ถูกต้องก่อนใช้งานต่อ

### 21.7 ยาติดหรือไม่ตกลงถ้วย

สาเหตุที่เป็นไปได้:

- ช่องยาแน่นเกินไป
- กล่องยาหรือเม็ดยามีขนาดไม่เหมาะ
- มุมเปิด servo ไม่พอ
- ทางปล่อยยาไม่เรียบ
- ถ้วยไม่ตรงตำแหน่ง

วิธีแก้:

1. ปิดเครื่อง
2. นำยาออกจากช่องที่ติด
3. ตรวจสอบสิ่งกีดขวาง
4. ปรับ Servo Calibration
5. ตรวจสอบถ้วยรับยา
6. ทดสอบด้วยวัตถุทดสอบก่อนใส่ยาจริง

### 21.8 Buzzer ไม่ดัง

สาเหตุที่เป็นไปได้:

- Alert Buzzer ถูกปิด
- Buzzer module หลุดหรือเสีย
- ตั้งค่าขั้ว active high/low ไม่ตรง
- เครื่องไม่ได้เข้าสถานะเตือนจริง

วิธีแก้:

1. เปิด Device Alert Settings
2. ตรวจสอบว่า Alert Buzzer เปิดอยู่
3. กดปุ่มค้างประมาณ 2 วินาทีตอนไม่มีรอบยาค้างเพื่อทดสอบ
4. หากไม่ดัง ให้ผู้ดูแลตรวจสายหรือ module

### 21.9 เสียง Audio ไม่เล่น

สาเหตุที่เป็นไปได้:

- Alert Audio ถูกปิด
- ไม่มีไฟล์เสียง active
- ไฟล์เสียงยังไม่ sync ลงเครื่อง
- SD Card ไม่พร้อม
- Speaker/DAC มีปัญหา

วิธีแก้:

1. เปิด Device Alert Settings
2. ตรวจสอบ Audio Files ใน Dashboard
3. สั่ง sync_audio หากมีปุ่มหรือคำสั่งรองรับ
4. ทดสอบ play_audio
5. ตรวจสอบ SD Card และ speaker

### 21.10 เครื่องรีสตาร์ตตอน servo ทำงาน

สาเหตุที่เป็นไปได้:

- ไฟเลี้ยง servo ไม่พอ
- สายไฟหลวม
- servo ฝืนกลไก
- ใช้ไฟจาก ESP32 ไปเลี้ยง servo โดยตรง

วิธีแก้:

1. หยุดใช้งานเครื่อง
2. ตรวจสอบ power supply servo
3. ตรวจสอบว่ามี GND ร่วมถูกต้อง
4. ลดแรงฝืนของกลไก
5. ทดสอบ servo ทีละช่อง
6. ให้ช่างตรวจสอบระบบไฟ

### 21.11 คำสั่งจาก Dashboard ตอบสนองช้า

สาเหตุที่เป็นไปได้:

- MQTT ไม่เชื่อมต่อ
- WebSocket ไม่เชื่อมต่อ
- เครื่องใช้ HTTP polling fallback
- Wi-Fi สัญญาณอ่อน
- Backend มีภาระสูง

วิธีแก้:

1. ดู OLED ว่า MQTT เป็น OK หรือไม่
2. เปิด Diagnostics
3. ตรวจสอบ MQTT Pipeline
4. ตรวจสอบ Wi-Fi RSSI ในหน้า Devices
5. สั่ง reload_schedule แล้วดู Command History
6. หากช้าต่อเนื่อง ให้ตรวจเครือข่าย

## 22. การใช้งานเมื่ออินเทอร์เน็ตหรือ Server มีปัญหา

ข้อจำกัดเมื่อ offline:

- Dashboard/Mobile App อาจยังไม่เห็นสถานะล่าสุด

- การแก้ไขตารางใหม่จะยังไม่เข้าถึงเครื่องจนกว่าเครื่อง online

สิ่งที่ควรทำ:

1. ตรวจสอบว่าเครื่องยังมีไฟ
2. ตรวจสอบ Wi-Fi
3. ตรวจสอบ OLED
4. ให้ผู้ใช้รับยาตามเสียง/ไฟเตือนของเครื่อง
5. เมื่อระบบกลับมา online ให้ตรวจสอบ Dose Events และ History

## 23. การรีเซ็ตหรือตั้งค่าเครื่องใหม่

### 23.1 กรณีที่ควรตั้งค่าใหม่

- เปลี่ยน Wi-Fi
- ย้ายเครื่องไปสถานที่ใหม่
- Device Token หมดอายุหรือถูกสร้างใหม่
- API URL เปลี่ยน
- เครื่องจำค่าผิด
- Dashboard เห็นเครื่อง offline ทั้งที่ Wi-Fi ปกติ

### 23.2 วิธีที่แนะนำ

1. สำรองข้อมูลหรือจดชื่ออุปกรณ์
2. ตรวจสอบตารางยาและช่องยา
3. ให้ Admin สั่ง reset_config หรือ clear Wi-Fi ตามเครื่องมือที่มี
4. รอเครื่อง reboot
5. รอ Pairing Mode
6. จับคู่ใหม่ผ่าน Dashboard หรือ Mobile App
7. ตรวจสอบว่าเครื่อง online
8. สั่ง reload_schedule
9. ทดสอบการเตือนและการจ่ายยา

### 23.3 หลัง reset ต้องตรวจอะไรบ้าง

- Wi-Fi connected
- API URL ถูกต้อง
- Device Token ถูกต้อง
- อุปกรณ์ online ใน Dashboard
- Device Slots ยังถูกต้อง
- Servo Calibration ยังถูกต้อง
- Schedules active
- แจ้งเตือนทำงาน

## 24. ภาคผนวกสำหรับผู้ดูแลหรือช่างเทคนิค

หัวข้อนี้เหมาะสำหรับผู้ดูแลระบบหรือช่างที่ต้องตรวจสอบตัวเครื่อง ไม่ใช่ขั้นตอนที่ผู้ป่วยต้องทำเอง

### 24.1 สรุปอุปกรณ์ภายใน

| อุปกรณ์ | รายละเอียด |
| --- | --- |
| MCU | ESP32 38-pin ESP-WROOM-32 NodeMCU CP2102 |
| Servo Driver | PCA9685 16-channel PWM |
| Servo | MG90S 180 degree จำนวน 7 ตัว |
| Audio DAC | PCM5102A GY-PCM5102 I2S |
| Storage | microSD Card สำหรับไฟล์เสียง |
| Buzzer | GPIO32 |
| Confirm Button | GPIO34 |
| Button LED | GPIO33 |
| Pairing LED | GPIO2 |
| OLED | I2C 128x64 ที่ address 0x3C หรือ 0x3D |

### 24.2 Pinout สำคัญ

| หน้าที่ | GPIO |
| --- | --- |
| I2C SDA สำหรับ OLED/PCA9685 | GPIO21 |
| I2C SCL สำหรับ OLED/PCA9685 | GPIO22 |
| I2S DATA ไป PCM5102A | GPIO25 |
| I2S BCLK ไป PCM5102A | GPIO26 |
| I2S LRCK ไป PCM5102A | GPIO27 |

| Buzzer | GPIO32 |
| Pairing LED | GPIO2 |

### 24.3 กฎไฟเลี้ยง

- Servo ต้องใช้ไฟเลี้ยงแยกจาก ESP32
- ต้องต่อ GND ร่วมกันทุก module
- ห้ามให้ servo กินไฟจากขา 3V3 หรือ 5V ของ ESP32 โดยตรง
- ควรมี capacitor ประมาณ 1000uF ที่รางไฟ servo
- หาก ESP32 reset ตอน servo ขยับ ให้สงสัยไฟ servo ตกก่อน

### 24.4 คำสั่ง Serial ที่ใช้ตรวจสอบ

Serial Monitor: 115200 baud

คำสั่งที่ใช้บ่อย:

```text
help
config
wifi-scan
wifi-set <ssid>|<password>
wifi-clear
wifi-connect
api-set <url>
token-set <device-token>
i2c-scan
ws-status
mqtt-status
servo <channel> <angle>
sd-test
buzzer-test
tone-test
audio-test [filename.wav]
audio-sync
button-test
hardware-test
confirm-pending
reboot
```

ตัวอย่างทดสอบ servo:

```text
servo 0 30
servo 0 140
```

หมายเหตุ:

- Channel 0 คือช่องยา 1
- Channel 6 คือช่องยา 7
- ควรถอดยาออกก่อนทดสอบ

### 24.5 ตรวจสอบ I2C

ใช้คำสั่ง:

```text
i2c-scan
```

ค่าที่คาดหวัง:

- PCA9685: `0x40`
- OLED: `0x3C` หรือ `0x3D`

หากไม่พบ:

- ตรวจสาย SDA/SCL
- ตรวจไฟ 3V3
- ตรวจ GND ร่วม
- ตรวจ address ของ module

### 24.6 ตรวจสอบ SD และเสียง

SD Card ควรเป็น FAT32 และไฟล์เสียงควรอยู่ในตำแหน่งที่ firmware รองรับ เช่น:

```text
/sound1.wav
/sound2.wav
/audio/<filename>.wav
```

รูปแบบเสียงที่แนะนำ:

- WAV PCM
- 16-bit
- mono หรือ stereo
- 24000 Hz หรือ 44100 Hz

คำสั่งตรวจสอบ:

```text
sd-test
audio-test sound1.wav
audio-sync
```

### 24.7 ตรวจสอบปุ่มรับยา

ใช้คำสั่ง:

```text
button-test
```

สิ่งที่ควรเห็น:

- ค่าดิบของ GPIO34 เปลี่ยนเมื่อกดปุ่ม
- firmware ตรวจจับ polarity ได้ถูกต้อง
- GPIO34 ต้องมี pull-up หรือ pull-down ภายนอก เพราะไม่มี internal pull resistor ที่เชื่อถือได้
