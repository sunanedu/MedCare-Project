# คู่มือการใช้งาน MedCare Mobile Application

วันที่จัดทำ: 17 มิถุนายน 2026  
แอปพลิเคชัน: `com.medcare.mobile`  
เวอร์ชันแอป: `0.0.1`  
ระบบที่เกี่ยวข้อง: MedCare Mobile, MedCare Backend API, กล่องจ่ายยา MedCare ESP32, Dashboard/Web, MQTT/WebSocket

## 1. วัตถุประสงค์ของคู่มือ

คู่มือนี้อธิบายการใช้งานโมบายแอปพลิเคชัน MedCare ตั้งแต่ขั้นตอนแรกจนถึงการใช้งานประจำวัน ครอบคลุมการตั้งค่าเซิร์ฟเวอร์ การสมัครและเข้าสู่ระบบ การจับคู่อุปกรณ์ การเพิ่มยา การกำหนดช่องยา การตั้งตารางเวลา การตั้งค่าการแจ้งเตือน การดูประวัติ และการตรวจสอบปัญหาเบื้องต้น

คู่มือนี้เหมาะสำหรับผู้ใช้ 3 กลุ่มหลัก:

- ผู้ดูแลหลักของผู้ป่วยหรือเจ้าของบัญชี
- ผู้ช่วยดูแลหรือสมาชิกครอบครัว
- ผู้ใช้งานทั่วไปที่ต้องการดูสถานะยาและการแจ้งเตือน

## 2. ภาพรวมของระบบ MedCare Mobile

MedCare Mobile เป็นแอปสำหรับควบคุมและติดตามการจ่ายยาจากกล่องยาอัจฉริยะ โดยแอปทำงานร่วมกับ Backend API และอุปกรณ์ ESP32 เพื่อจัดการยา ตารางเวลา และการแจ้งเตือน

เมนูหลักของแอปมี 4 ส่วน:

| เมนู | ใช้ทำอะไร |
|---|---|
| Home | ดูภาพรวมวันนี้ สถานะอุปกรณ์ ยาที่ถึงเวลา ยาถัดไป และสถิติการกินยา |
| Schedule | จัดการตารางเวลา รายการยา และประวัติการกินยา |
| Notifications | ดูการแจ้งเตือนทั้งหมด กรองแจ้งเตือน และทำเครื่องหมายว่าอ่านแล้ว |
| Settings | ตั้งค่าบัญชี อุปกรณ์ ช่องยา เสียง แจ้งเตือน ครอบครัว เซิร์ฟเวอร์ และระบบวิเคราะห์ปัญหา |

เวลาที่ใช้ในระบบอ้างอิงโซนเวลา `Asia/Bangkok` เป็นหลัก

## 3. สิ่งที่ต้องเตรียมก่อนเริ่มใช้งาน

ก่อนเริ่มใช้งานจริง ควรเตรียมสิ่งต่อไปนี้ให้พร้อม:

- โทรศัพท์มือถือที่ติดตั้งแอป MedCare Mobile แล้ว
- อินเทอร์เน็ตบนมือถือหรือ Wi-Fi ที่ใช้งานได้
- บัญชีผู้ใช้ MedCare หรืออีเมลสำหรับสมัครบัญชีใหม่
- กล่องจ่ายยา MedCare ที่เปิดเครื่องอยู่
- Bluetooth ของโทรศัพท์มือถือเปิดใช้งาน สำหรับขั้นตอนจับคู่อุปกรณ์
- Wi-Fi สำหรับให้อุปกรณ์ ESP32 เชื่อมต่อ โดยควรเป็น Wi-Fi 2.4 GHz
- รหัสผ่าน Wi-Fi ที่ถูกต้อง
- URL ของ Backend API ที่ต้องการใช้งาน เช่น Production หรือเซิร์ฟเวอร์ภายใน
- ยาและข้อมูลยาที่ต้องการบันทึก เช่น ชื่อยา จำนวน หน่วย ช่องยา และคำแนะนำ

ข้อควรระวัง:

- อย่าใส่ยาเข้าช่องก่อนทดสอบการเปิดปิดช่องยาและการปรับ servo เสร็จ
- ตรวจสอบให้แน่ใจว่าช่องยาจริงตรงกับช่องที่ตั้งค่าในแอป
- ถ้าใช้งาน Production ให้ใช้ HTTPS URL ของระบบจริงเป็นค่าเริ่มต้น
- ถ้าใช้งานเซิร์ฟเวอร์ภายใน ต้องแน่ใจว่าโทรศัพท์และอุปกรณ์ ESP32 เข้าถึงเซิร์ฟเวอร์นั้นได้จริง

## 4. สิทธิ์ผู้ใช้และบทบาทในระบบ

ระบบมีบทบาทผู้ใช้หลายระดับ สิทธิ์ที่เห็นในแอปอาจแตกต่างกันตามบทบาท

| บทบาท | สิทธิ์หลัก |
|---|---|
| Owner | จัดการได้ทั้งหมด รวมถึงสมาชิกครอบครัว อุปกรณ์ ตารางยา แจ้งเตือน และ Audit Logs |
| Admin | จัดการส่วนสำคัญได้ใกล้เคียง Owner เช่น สมาชิก แจ้งเตือน และ Audit Logs |
| Helper | ช่วยดูแลการใช้งานประจำวัน เช่น ดู/จัดการยา ตาราง และอุปกรณ์บางส่วนตามสิทธิ์ |
| Viewer | ดูข้อมูลได้เป็นหลัก แต่ไม่สามารถแก้ไขหรือลบข้อมูลสำคัญได้ |

ถ้าพบว่าปุ่มแก้ไข ปุ่มลบ หรือปุ่มบันทึกกดไม่ได้ ให้ตรวจสอบบทบาทผู้ใช้ก่อน เพราะบัญชีแบบ Viewer จะถูกจำกัดสิทธิ์ไว้เพื่อป้องกันการแก้ไขโดยไม่ตั้งใจ

## 5. การเปิดแอปครั้งแรก

เมื่อเปิดแอปครั้งแรก ให้ทำตามลำดับนี้:

1. เปิดแอป MedCare Mobile
2. ตรวจสอบว่าโทรศัพท์เชื่อมต่ออินเทอร์เน็ตแล้ว
3. ถ้าเป็นเครื่องใหม่หรือยังไม่เคยตั้งค่า ให้ตรวจสอบ Server Settings ก่อน
4. สมัครบัญชีใหม่หรือเข้าสู่ระบบด้วยบัญชีที่มีอยู่
5. เมื่อเข้าสู่ระบบสำเร็จ แอปจะแสดงหน้า Home

หากแอปโหลดข้อมูลไม่ได้ตั้งแต่หน้าแรก ให้ตรวจสอบ Server Settings และทดสอบการเชื่อมต่อก่อนเข้าสู่ระบบ

## 6. การตั้งค่าเซิร์ฟเวอร์ก่อนเข้าสู่ระบบ

หน้าตั้งค่าเซิร์ฟเวอร์ใช้กำหนดว่าแอปจะเชื่อมต่อ Backend API ที่ใด

### 6.1 เข้าเมนู Server Settings

จากหน้า Login:

1. แตะไอคอนตั้งค่าเซิร์ฟเวอร์ที่มุมขวาบน
2. แอปจะเปิดหน้า Server Settings
3. เลือกโปรไฟล์ API ที่ต้องการใช้งาน
4. กด Test Connection
5. ถ้าทดสอบสำเร็จ ให้กด Save
6. กลับไปหน้า Login แล้วเข้าสู่ระบบ

### 6.2 โปรไฟล์เซิร์ฟเวอร์ที่มีในแอป

| โปรไฟล์ | URL |
|---|---|
| Production | `https://medcare.i-esaan.com/api/v1` |
| Work Office | `http://192.168.20.225:3001/api/v1` |
| Home Lab | `http://192.168.1.121:3001/api/v1` |
| Local Device / USB debug | `http://127.0.0.1:3001/api/v1` |
| Custom | กำหนด URL เอง |

### 6.3 เงื่อนไขของ API URL

URL ที่บันทึกต้องเป็นไปตามเงื่อนไขนี้:

- ต้องขึ้นต้นด้วย `http://` หรือ `https://`
- ต้องลงท้ายด้วย `/api/v1`
- ถ้าใช้งาน Production แนะนำให้ใช้ `https://medcare.i-esaan.com/api/v1`
- ถ้าใช้งานกับ ESP32 ผ่านเครือข่ายจริง หลีกเลี่ยงการใช้ `127.0.0.1` เป็น URL ของอุปกรณ์ เพราะ ESP32 จะมอง `127.0.0.1` เป็นตัวอุปกรณ์เอง ไม่ใช่เครื่องเซิร์ฟเวอร์

### 6.4 การทดสอบการเชื่อมต่อ

เมื่อกด Test Connection แอปจะเรียก `/health` ของ API:

- ถ้าสำเร็จ แปลว่าแอปติดต่อ Backend ได้
- ถ้าล้มเหลว ให้ตรวจสอบ URL, อินเทอร์เน็ต, VPN, Wi-Fi, firewall หรือสถานะเซิร์ฟเวอร์

## 7. การสมัครบัญชีใหม่

ใช้เมื่อต้องสร้างบัญชีผู้ใช้ MedCare เป็นครั้งแรก

ขั้นตอน:

1. ที่หน้า Login แตะ Register หรือ Create Account
2. กรอกชื่อ-นามสกุล
3. กรอกอีเมล
4. กรอกรหัสผ่าน
5. กรอกยืนยันรหัสผ่าน
6. กด Submit/Register
7. ถ้าข้อมูลถูกต้อง แอปจะสร้างบัญชีและเข้าสู่ระบบหรือพาไปขั้นตอนถัดไปตามการตั้งค่าระบบ

เงื่อนไขรหัสผ่าน:

- ต้องมีอย่างน้อย 8 ตัวอักษร
- ต้องมีตัวอักษรภาษาอังกฤษอย่างน้อย 1 ตัว
- ต้องมีตัวเลขอย่างน้อย 1 ตัว
- รหัสผ่านและยืนยันรหัสผ่านต้องตรงกัน

ข้อผิดพลาดที่อาจพบ:

| ข้อความ/อาการ | วิธีแก้ |
|---|---|
| อีเมลไม่ถูกต้อง | ตรวจสอบรูปแบบอีเมล เช่น `name@example.com` |
| อีเมลถูกใช้แล้ว | ใช้หน้า Login หรือ Forgot Password แทน |
| รหัสผ่านไม่ผ่านเงื่อนไข | ตั้งรหัสผ่านใหม่ให้มีตัวอักษรและตัวเลข |
| กรอกข้อมูลไม่ครบ | ตรวจสอบทุกช่องก่อนส่ง |

## 8. การเข้าสู่ระบบ

ขั้นตอน:

1. เปิดหน้า Login
2. กรอกอีเมล
3. กรอกรหัสผ่าน
4. กด Login
5. รอให้แอปตรวจสอบบัญชี
6. เมื่อสำเร็จ แอปจะเข้าสู่หน้า Home

ถ้าเข้าสู่ระบบไม่ได้:

- ตรวจสอบ Server Settings ว่าเลือกเซิร์ฟเวอร์ถูกต้อง
- ตรวจสอบอีเมลและรหัสผ่าน
- ใช้ Forgot Password หากจำรหัสผ่านไม่ได้
- หากบัญชีถูกปิดใช้งาน ให้ติดต่อผู้ดูแลระบบ

## 9. การลืมรหัสผ่าน

ขั้นตอน:

1. ที่หน้า Login แตะ Forgot Password
2. กรอกอีเมลที่ใช้สมัคร
3. กด Submit
4. ระบบจะส่งคำขอรีเซ็ตรหัสผ่านตามช่องทางที่ Backend กำหนด
5. ทำตามขั้นตอนในอีเมลหรือช่องทางที่ได้รับ

หากไม่ได้รับอีเมล:

- ตรวจสอบว่าอีเมลถูกต้อง
- ตรวจสอบกล่อง Spam/Junk
- ตรวจสอบว่าแอปเชื่อมต่อเซิร์ฟเวอร์ที่ถูกต้อง
- ติดต่อผู้ดูแลระบบหากยังไม่สำเร็จ

## 10. หน้า Home

หน้า Home เป็นหน้าหลักสำหรับดูสถานะประจำวัน

ข้อมูลที่แสดง:

- คำทักทายผู้ใช้
- สถานะอุปกรณ์ เช่น online/offline
- เวลาที่อุปกรณ์ติดต่อระบบล่าสุด
- ยาที่ถึงเวลาหรือใกล้ถึงเวลา
- ยาถัดไปที่ต้องกิน
- จำนวนรายการที่กินแล้ว
- จำนวนรายการที่รอดำเนินการ
- เปอร์เซ็นต์ adherence ของวันนี้
- แผนการกินยาวันนี้

การใช้งาน:

1. เปิดหน้า Home หลังเข้าสู่ระบบ
2. ตรวจสอบว่าสถานะอุปกรณ์เป็น online
3. ดูแถบแจ้งเตือนยาที่ถึงเวลา
4. ดูรายการยาถัดไป
5. แตะ View Schedules เพื่อไปจัดการตารางยา
6. แตะ My Devices เพื่อไปดูอุปกรณ์

คำแนะนำ:

- หากอุปกรณ์ offline ให้ตรวจสอบไฟเลี้ยง Wi-Fi และหน้า Device Diagnostics
- หากตารางวันนี้ไม่แสดง ให้ตรวจสอบว่าตั้ง active days ถูกต้องหรือไม่
- หน้า Home มีการ refresh ข้อมูลเป็นระยะ จึงอาจต้องรอสักครู่หลังเปลี่ยนข้อมูล

## 11. การจับคู่อุปกรณ์ MedCare

การจับคู่อุปกรณ์คือการเชื่อมกล่องจ่ายยาเข้ากับบัญชีและเซิร์ฟเวอร์

### 11.1 เตรียมอุปกรณ์ก่อนจับคู่

ก่อนเริ่ม:

1. เสียบไฟกล่องจ่ายยา MedCare
2. เปิด Bluetooth บนโทรศัพท์
3. เปิด Wi-Fi ที่ต้องการให้อุปกรณ์ใช้งาน
4. นำโทรศัพท์ไว้ใกล้กล่องยา
5. ตรวจสอบว่าอุปกรณ์อยู่ในโหมด pairing
6. เตรียม SSID และรหัสผ่าน Wi-Fi
7. ตรวจสอบ API URL ที่อุปกรณ์ต้องเชื่อมต่อ

ถ้าอุปกรณ์เคยจับคู่กับระบบอื่นมาก่อน อาจต้อง Reset Config จากเมนู My Devices หรือกด reset ตามวิธีของอุปกรณ์ก่อน

### 11.2 เข้าเมนู Pair Device

ขั้นตอน:

1. ไปที่ Settings
2. แตะ Pair Device
3. อนุญาตสิทธิ์ Bluetooth/Nearby Devices/Location หากระบบถาม
4. กดค้นหาอุปกรณ์
5. เลือกอุปกรณ์ที่ขึ้นต้นด้วยชื่อ `MedCare-`
6. รอให้แอปเชื่อมต่อผ่าน Bluetooth

### 11.3 กรอกข้อมูลอุปกรณ์

หลังเชื่อมต่อ Bluetooth แล้ว ให้กรอกหรือตรวจสอบข้อมูล:

- Device Name: ชื่ออุปกรณ์ เช่น `กล่องยาคุณแม่`
- MAC Address: ระบบอาจอ่านให้อัตโนมัติ ถ้าไม่ขึ้นให้กรอกเองตามฉลากหรือข้อมูลจากอุปกรณ์
- Wi-Fi SSID: ชื่อ Wi-Fi ที่อุปกรณ์ต้องเชื่อมต่อ
- Wi-Fi Password: รหัสผ่าน Wi-Fi
- Service URL/API Base URL: URL ที่ ESP32 จะใช้ติดต่อ Backend API

ตัวอย่าง Service URL:

- Production: `https://medcare.i-esaan.com/api/v1`
- เซิร์ฟเวอร์ในออฟฟิศ: `http://192.168.20.225:3001/api/v1`
- เซิร์ฟเวอร์ในบ้าน: `http://192.168.1.121:3001/api/v1`

ข้อควรระวัง:

- URL ต้องเป็น URL ที่อุปกรณ์ ESP32 เข้าถึงได้ ไม่ใช่เข้าถึงได้เฉพาะในโทรศัพท์
- ห้ามใช้ `127.0.0.1` กับ ESP32 เว้นแต่เข้าใจการ debug เครือข่ายอย่างชัดเจน
- หากใช้ HTTPS ต้องแน่ใจว่า certificate ของระบบใช้งานได้

### 11.4 ส่งค่าการตั้งค่าเข้าอุปกรณ์

ขั้นตอน:

1. ตรวจสอบข้อมูลทั้งหมดอีกครั้ง
2. กดบันทึกหรือส่งค่า provisioning
3. แอปจะสร้างหรืออัปเดตอุปกรณ์บน Backend
4. Backend จะออก device token ให้
5. แอปจะส่ง Wi-Fi, API URL และ device token ไปยัง ESP32 ผ่าน BLE
6. รอให้อุปกรณ์เชื่อมต่อ Wi-Fi และ Backend

สถานะที่อาจเห็น:

| สถานะ | ความหมาย |
|---|---|
| waiting_for_wifi | อุปกรณ์กำลังพยายามเชื่อมต่อ Wi-Fi |
| wifi_connected | เชื่อมต่อ Wi-Fi สำเร็จ |
| wifi_failed | เชื่อมต่อ Wi-Fi ไม่สำเร็จ |
| saved_reconnecting | บันทึกค่าแล้วและกำลัง reconnect |
| wifi_cleared | ล้างค่าการเชื่อมต่อแล้ว |
| pairing_window | อุปกรณ์เปิดช่วงจับคู่ |

### 11.5 ตรวจสอบหลังจับคู่สำเร็จ

หลังจับคู่เสร็จ:

1. ไปที่ Home
2. ตรวจสอบว่าอุปกรณ์เป็น online
3. ไปที่ Settings > My Devices
4. ตรวจสอบชื่ออุปกรณ์, MAC, IP, Firmware และ Last seen
5. หากต้องการตรวจเชิงลึก ให้ไปที่ Device Diagnostics

## 12. การจัดการอุปกรณ์

Settings > My Devices

ข้อมูลที่แสดง:

- จำนวนอุปกรณ์ทั้งหมด
- จำนวนอุปกรณ์ online
- รายชื่ออุปกรณ์

## 13. การเพิ่มรายการยา

- หากลบยาที่กำลังถูกใช้ใน schedule อาจทำให้ตารางเวลาบางรายการไม่สมบูรณ์

## 15. การกำหนดช่องยาในอุปกรณ์

ช่องยาใช้เชื่อมระหว่างรายการยาในระบบกับช่องจริงในกล่องยา

เข้าเมนู:

Settings > Device Slots

### 15.1 การตั้งค่าช่องยา

ขั้นตอน:

1. ไปที่ Settings
2. แตะ Device Slots
3. ถ้ามีหลายอุปกรณ์ ให้เลือกอุปกรณ์ที่ต้องการ
4. ระบบจะแสดงช่องยา 1-7
5. เปิดหรือปิดใช้งานแต่ละช่อง
6. เลือกยาที่ต้องการผูกกับช่องนั้น
7. บันทึกการเปลี่ยนแปลง

### 15.2 หลักการผูกช่องยา

ตัวอย่าง:

| ช่องจริง | ตั้งค่าในแอป |
|---|---|
| ช่อง 1 ใส่ Paracetamol | Slot 1 = Paracetamol |
| ช่อง 2 ใส่ Vitamin C | Slot 2 = Vitamin C |
| ช่อง 3 ว่าง | Slot 3 = Empty หรือ Disabled |

คำแนะนำ:

- หลังเติมยา ให้ตรวจสอบในแอปทุกครั้งว่าช่องตรงกับยาจริง
- ถ้าช่องใดไม่ใช้ ให้ตั้งเป็น Empty หรือปิดใช้งาน
- ถ้าเปลี่ยนตำแหน่งยาในกล่อง ให้กลับมาอัปเดต Device Slots ทันที

## 16. การปรับเทียบ Servo ของช่องยา

Servo Calibration ใช้ปรับมุมเปิด-ปิดของแต่ละช่องยา เพื่อให้อุปกรณ์ปล่อยยาได้ถูกต้อง

เข้าเมนู:

Settings > Servo Calibration

### 16.1 ข้อควรระวังก่อนปรับ servo

ก่อนปรับ:

- นำยาออกจากช่องก่อนทดสอบ
- วางอุปกรณ์บนพื้นเรียบ
- อย่าเอานิ้วหรือสิ่งของไปขวางกลไก
- ทดสอบทีละช่อง
- หากไม่แน่ใจ ให้ปิด Live Preview ก่อน

### 16.2 ขั้นตอนการปรับเทียบ

1. ไปที่ Settings > Servo Calibration
2. ถ้ามีหลายอุปกรณ์ ให้เลือกอุปกรณ์ที่ต้องการ
3. ตรวจสอบสถานะอุปกรณ์ว่า online
4. เลือกช่องที่ต้องการปรับ
5. ปรับค่า Closed/Lock Angle สำหรับตำแหน่งปิด
6. ปรับค่า Open/Release Angle สำหรับตำแหน่งเปิด
7. กด Test Closed เพื่อทดสอบตำแหน่งปิด
8. กด Test Open เพื่อทดสอบตำแหน่งเปิด
9. กด Test 2s เพื่อทดสอบรอบเปิดค้างและปิดกลับ
10. หาก servo ติดตั้งกลับด้าน ให้เปิดตัวเลือก Reverse สำหรับช่องนั้น
11. เมื่อทุกช่องถูกต้อง ให้กด Save

ค่ามุม servo:

- ปรับได้ตั้งแต่ 0-180 องศา
- ค่าเริ่มต้นที่ใช้ทั่วไปคือ Closed ประมาณ 30 และ Open ประมาณ 140
- หากช่องเปิดผิดทิศ ให้ลองใช้ Reverse แทนการเดาสุ่มค่ามุมหลายรอบ

หลังบันทึก ระบบจะส่งคำสั่ง reload schedule ไปที่อุปกรณ์ เพื่อให้อุปกรณ์ใช้ค่าล่าสุด

## 17. การตั้งค่าเสียงแจ้งเตือน

เสียงแจ้งเตือนใช้ประกอบตารางจ่ายยา เช่น เสียงเตือนเมื่อถึงเวลากินยา

เข้าเมนู:

Settings > Audio Files

### 17.1 การอัปโหลดไฟล์เสียง

ขั้นตอน:

1. ไปที่ Settings > Audio Files
2. แตะ Upload
3. เลือกไฟล์เสียงจากโทรศัพท์
4. รอให้ระบบอัปโหลด
5. ตรวจสอบว่าไฟล์ขึ้นในรายการ
6. กด Preview เพื่อฟังตัวอย่าง
7. เปิด Active หากต้องการให้ใช้ไฟล์นี้ได้

ระบบแสดงข้อมูลไฟล์ เช่น ชื่อไฟล์ format, duration, ขนาดไฟล์ และเวลาที่แก้ไขล่าสุด

### 17.2 การสร้างเสียงด้วย AI/TTS

หากระบบตั้งค่า Google TTS provider แล้ว สามารถสร้างเสียงจากข้อความได้

ขั้นตอน:

1. ไปที่ Settings > Audio Files
2. ตรวจสอบว่า provider พร้อมใช้งาน
3. กรอกชื่อเสียง
4. กรอกข้อความที่ต้องการให้พูด
5. เลือก voice เช่น Kore, Puck, Charon, Fenrir, Aoede, Leda, Orus หรือ Zephyr
6. กด Generate
7. ฟัง Preview
8. ตั้ง Active หากต้องการใช้

หาก provider ไม่พร้อมใช้งาน ให้ติดต่อผู้ดูแลระบบ Backend เพื่อกำหนดค่า TTS

### 17.3 การซิงก์เสียงไปยังอุปกรณ์

หลังเพิ่มหรือแก้ไขเสียง:

1. กด Sync Audio
2. ระบบจะส่งคำสั่ง `sync_audio` ไปยังอุปกรณ์ทั้งหมด
3. รอให้อุปกรณ์ดาวน์โหลดหรืออัปเดตไฟล์เสียง
4. หากเสียงยังไม่เล่น ให้ตรวจสอบ Device Diagnostics

## 18. การตั้งค่าเสียงและการแจ้งเตือนบนอุปกรณ์

เข้าเมนู:

Settings > Device Alert Settings

ตั้งค่าได้ดังนี้:

- เปิด/ปิด buzzer
- เปิด/ปิด extra audio
- ตั้ง confirm timeout ตั้งแต่ 1-60 นาที
- เลือก preset เช่น 5, 10, 15, 30 หรือ 60 นาที

ความหมายของ confirm timeout:

- เมื่อถึงเวลาจ่ายยา อุปกรณ์จะรอการยืนยันภายในเวลาที่กำหนด
- หากไม่มีการยืนยัน ระบบสามารถนับเป็น missed หรือเข้าสู่กระบวนการแจ้งเตือนซ้ำตามนโยบาย notification

คำแนะนำ:

- ผู้สูงอายุที่ได้ยินเสียงยาก ควรเปิด buzzer และ extra audio
- หากผู้ใช้มักยืนยันช้า ให้ตั้ง timeout นานขึ้น
- หากต้องการควบคุมเข้มงวด ให้ตั้ง timeout สั้นลงและเปิด escalation

## 19. การตั้งค่านโยบายแจ้งเตือน

เข้าเมนู:

Settings > Notification Settings

เมนูนี้เหมาะสำหรับ Owner/Admin เพื่อกำหนดวิธีแจ้งเตือนเมื่อมีเหตุการณ์สำคัญ

### 19.1 ตัวเลือกหลัก

ตั้งค่าได้ดังนี้:

- Escalation on missed doses: แจ้งเตือนซ้ำหรือแจ้งผู้ดูแลเมื่อพลาดยา
- Repeat until confirmed: แจ้งเตือนซ้ำจนกว่าจะยืนยัน
- Repeat interval: รอบการแจ้งซ้ำ เช่น 5, 10, 15, 30, 60 นาที
- Max repeats: จำนวนครั้งสูงสุด เช่น 1, 2, 3, 5, 10 ครั้ง
- Quiet hours: ช่วงเวลางดแจ้งเตือน

### 19.2 ช่องทางแจ้งเตือน

ช่องทางที่รองรับ:

| ช่องทาง | รายละเอียด |
|---|---|
| In-app | แจ้งในแอป |
| Push | แจ้งเตือนผ่านระบบมือถือ |
| Email | ต้องกรอกอีเมลปลายทาง |
| LINE | ต้องมี channel access token และ target ID |
| Telegram | ต้องมี bot token และ chat/channel ID |

ระบบจะตรวจสอบรูปแบบข้อมูลก่อนบันทึก เช่น รูปแบบเวลา อีเมล Telegram token และ LINE configuration

### 19.3 Quiet Hours

Quiet Hours ใช้ลดการรบกวนในช่วงเวลาที่กำหนด

ตัวอย่าง:

- เปิด Quiet Hours
- Start: `22:00`
- End: `06:00`

คำแนะนำ:

- ถ้าเป็นยาสำคัญที่ต้องแจ้งเตือนตลอดเวลา ไม่ควรตั้ง Quiet Hours ครอบเวลานั้น
- ถ้าเป็นแจ้งเตือนทั่วไป สามารถใช้ Quiet Hours เพื่อลดเสียงรบกวนกลางคืน

## 20. การสร้างตารางการกินยา

ตารางการกินยาเป็นหัวใจหลักของระบบ ควรตั้งหลังจากเพิ่มยา กำหนดช่องยา และปรับ servo แล้ว

3. เลือกเวลา โดยใช้ตัวเลือกชั่วโมง/นาที หรือ preset เช่น 09:00, 12:00, 16:00, 18:00

8. เลือก meal timing เช่น ก่อนอาหาร หลังอาหาร ก่อนนอน หรือเวลาใดก็ได้
9. เลือก sound file ถ้าต้องการใช้เสียงเฉพาะ
10. ใส่ note หากมี
11. กด Save

## 21. การดู แก้ไข และลบตารางยา

เข้าเมนู:

Schedule

### 21.1 ดูตาราง

หน้า Schedule จะแสดงแท็บวันจันทร์ถึงอาทิตย์ และเลือกวันปัจจุบันเป็นค่าเริ่มต้น

การใช้งาน:

1. แตะแท็บวันที่ต้องการดู
2. ตรวจสอบรายการตารางที่ active ในวันนั้น
3. ดูสถานะของแต่ละรายการ เช่น pending, due, taken หรือ missed
4. แตะรายการเพื่อแก้ไข

### 21.2 แก้ไขตาราง

1. แตะ schedule ที่ต้องการแก้ไข
2. เปลี่ยนเวลา ยา วัน ช่อง หรือเสียง
3. กด Save

### 21.3 ลบตาราง

1. ที่หน้า Schedule เลื่อนรายการเพื่อเปิด action ลบ หรือเข้าไปที่หน้ารายละเอียด
2. แตะ Delete
3. ยืนยันการลบ

ข้อควรระวัง:

- เมื่อลบ schedule แล้ว อุปกรณ์จะไม่จ่ายยาตามเวลานั้นอีก
- หลังแก้ schedule สำคัญ ควรตรวจ Device Diagnostics หรือรอให้อุปกรณ์ sync schedule ล่าสุด

5. เมื่อถึงเวลา ระบบจะส่งแจ้งเตือนในแอปและอุปกรณ์จะเตือนตามที่ตั้งค่า
6. ผู้ใช้รับประทานยาและยืนยันตามวิธีของอุปกรณ์
7. ตรวจสอบว่า status เปลี่ยนเป็น taken
8. ถ้ามียาที่ missed ให้เข้าไปตรวจที่ Notifications หรือ History

หากสถานะไม่เปลี่ยน:

- รอสักครู่เพื่อให้ระบบ sync
- ตรวจว่าอุปกรณ์ online หรือไม่
- ใช้ History เพื่อปรับสถานะรายการเป็น taken, missed หรือ cancelled ตามความจริง

## 34. Troubleshooting

### 34.1 เข้าสู่ระบบไม่ได้

สาเหตุที่เป็นไปได้:

- อีเมลหรือรหัสผ่านผิด
- Server Settings ชี้ผิดเซิร์ฟเวอร์
- อินเทอร์เน็ตไม่พร้อม
- บัญชีถูกปิดใช้งาน

วิธีแก้:

1. กด Server Settings แล้ว Test Connection
2. ตรวจอีเมลและรหัสผ่าน
3. ใช้ Forgot Password
4. ติดต่อผู้ดูแลหากบัญชี inactive

### 34.2 Test Connection ไม่ผ่าน

สาเหตุที่เป็นไปได้:

- URL ไม่ถูกต้อง
- URL ไม่ลงท้ายด้วย `/api/v1`
- เซิร์ฟเวอร์ปิดอยู่
- โทรศัพท์อยู่นอกเครือข่ายเดียวกับเซิร์ฟเวอร์ภายใน
- firewall หรือ VPN บล็อกการเชื่อมต่อ

วิธีแก้:

1. ตรวจ API URL
2. ลองใช้ Production หากไม่แน่ใจ
3. ปิด/เปิด Wi-Fi หรือ VPN
4. ตรวจสถานะ Backend

### 34.3 หาอุปกรณ์ Bluetooth ไม่เจอ

สาเหตุที่เป็นไปได้:

- Bluetooth ปิดอยู่
- ยังไม่ได้อนุญาตสิทธิ์ Bluetooth/Nearby Devices/Location
- อุปกรณ์ไม่ได้อยู่ใน pairing mode
- โทรศัพท์อยู่ไกลเกินไป
- อุปกรณ์เคยจับคู่แล้วและไม่ได้ reset config

วิธีแก้:

1. เปิด Bluetooth
2. อนุญาตสิทธิ์ทั้งหมดที่ระบบถาม
3. วางโทรศัพท์ใกล้อุปกรณ์
4. รีสตาร์ทอุปกรณ์
5. Reset Config แล้วลองจับคู่ใหม่

### 34.4 Wi-Fi failed ตอน pairing

สาเหตุที่เป็นไปได้:

- SSID ผิด
- รหัสผ่าน Wi-Fi ผิด
- Wi-Fi เป็น 5 GHz ที่ ESP32 ใช้ไม่ได้
- สัญญาณอ่อน
- router จำกัดอุปกรณ์ใหม่

วิธีแก้:

1. ตรวจชื่อ Wi-Fi และรหัสผ่าน
2. ใช้ Wi-Fi 2.4 GHz
3. ย้ายอุปกรณ์ใกล้ router
4. รีสตาร์ท router หากจำเป็น
5. Pair ใหม่

### 34.5 อุปกรณ์ offline

สาเหตุที่เป็นไปได้:

- ไฟเลี้ยงอุปกรณ์หลุด
- Wi-Fi หลุด
- Backend หรือ MQTT ไม่พร้อม
- API URL ในอุปกรณ์ผิด
- device token ไม่ถูกต้อง

วิธีแก้:

1. ตรวจไฟเลี้ยงอุปกรณ์
2. ตรวจ Wi-Fi
3. เปิด Settings > Device Diagnostics
4. กด refresh
5. กด reboot หากมีสิทธิ์
6. หากยังไม่หาย ให้ Reset Config และ Pair ใหม่

### 34.6 ถึงเวลาแล้วแต่ยาไม่จ่าย

ตรวจสอบตามลำดับ:

1. อุปกรณ์ online หรือไม่
2. Schedule เปิดใช้งานวันนั้นหรือไม่
3. เวลาที่ตั้งถูกต้องหรือไม่
4. Slot ที่ใช้ enabled หรือไม่
5. Slot ผูกกับยาถูกต้องหรือไม่
6. Servo เปิดปิดได้จริงหรือไม่
7. อุปกรณ์ได้รับ schedule ล่าสุดหรือไม่
8. ลองใช้ Device Diagnostics > reload_schedule
9. ตรวจ command history และ dispense logs

### 34.7 ยาออกผิดช่อง

วิธีแก้:

1. ตรวจ Device Slots ว่าผูกยากับช่องถูกต้อง
2. ตรวจ Medication ว่า primary slot ถูกต้อง
3. ตรวจ Schedule ว่าเลือก slot ถูกต้อง
4. ตรวจตำแหน่งยาจริงในกล่อง
5. ทดสอบ servo ทีละช่อง

### 34.8 แจ้งเตือนไม่เข้า

ตรวจสอบ:

1. Notification Settings เปิดช่องทางที่ต้องการหรือไม่
2. Quiet Hours ครอบเวลานั้นหรือไม่
3. โทรศัพท์อนุญาต push notification หรือไม่
4. Email/LINE/Telegram ตั้งค่าถูกต้องหรือไม่
5. อุปกรณ์ online หรือไม่
6. มี notification ในแท็บ Notifications หรือไม่

### 34.9 เสียงไม่เล่น

วิธีแก้:

1. ตรวจว่าไฟล์เสียง active
2. ตรวจว่า schedule เลือก sound file ถูกต้อง
3. ตรวจว่าเปิด extra audio ใน Device Alert Settings
4. กด Sync Audio
5. ตรวจ Device Diagnostics ว่าอุปกรณ์ online
6. ตรวจว่าลำโพงหรือ buzzer ของอุปกรณ์ทำงาน

### 34.10 ปุ่มแก้ไขกดไม่ได้

สาเหตุที่เป็นไปได้:

- บัญชีเป็น Viewer
- เมนูนั้นจำกัดเฉพาะ Owner/Admin
- ข้อมูลกำลัง loading หรือกำลังบันทึกอยู่

วิธีแก้:

1. ตรวจบทบาทผู้ใช้ในระบบ
2. ให้ Owner/Admin เปลี่ยนสิทธิ์
3. รอโหลดข้อมูลให้เสร็จแล้วลองใหม่

## 35. แนวทางความปลอดภัย

เพื่อความปลอดภัยของข้อมูลและผู้ป่วย:

- ห้ามแชร์รหัสผ่านบัญชี
- ห้ามส่ง device token ให้ผู้อื่น
- ใช้ Production HTTPS สำหรับระบบจริง
- จำกัดสิทธิ์สมาชิกตามความจำเป็น
- ใช้ Viewer สำหรับคนที่ต้องดูอย่างเดียว
- ตรวจ Audit Logs เมื่อมีการเปลี่ยนแปลงผิดปกติ
- ก่อนลบอุปกรณ์หรือ reset config ต้องแน่ใจว่ามีคนพร้อม pair ใหม่
- ตรวจช่องยาและตารางยาทุกครั้งหลังเติมยา
- ห้ามปรับ servo ขณะที่มีนิ้วหรือสิ่งของขวางกลไก

## 37. สรุปขั้นตอนสุดท้ายก่อนใช้งานจริง

ก่อนส่งมอบให้ผู้ใช้ ให้ตรวจให้ครบ:

- แอปเข้าสู่ระบบได้
- API เชื่อมต่อถูกต้อง
- อุปกรณ์ online
- ยาทุกตัวถูกเพิ่มในระบบ
- ช่องยาตรงกับยาจริง
- Servo เปิดปิดถูกต้องทุกช่องที่ใช้งาน
- เสียงแจ้งเตือนทำงาน
- Notification policy ตั้งค่าถูกต้อง
- Schedule ทำงานตามวันและเวลาที่ต้องการ
- History บันทึกผลได้
- ผู้ดูแลได้รับแจ้งเตือนเมื่อ missed dose
- สมาชิกครอบครัวมีสิทธิ์เหมาะสม
- ผู้ใช้เข้าใจวิธีกินยา ยืนยัน และแจ้งผู้ดูแลเมื่อมีปัญหา

เมื่อ checklist นี้ครบ ระบบ MedCare Mobile จะพร้อมสำหรับการใช้งานจริงตั้งแต่การติดตามรายวันไปจนถึงการตรวจสอบย้อนหลังและการดูแลโดยครอบครัว
